### 1-1. 영웅 기본 스탯 밸런스

#### 현황 진단 (DPS = ATK × ASPD)

| 영웅 | HP | ATK | ASPD | DPS | Range | 비고 |
|---|---|---|---|---|---|---|
| 광전사 | 2000 | 55 | 1.3 | **71.5** | 90 | 밸런스 양호 |
| 궁수 | 1300 | 40 | 1.3 | **52** | 420 | 원거리+크리15%로 실질 DPS 훨씬 높음 ⚠️ |

- **궁수**가 HP도 낮지만 원거리(420) + 크리 15% + 블링크로 실질 생존·딜 모두 1위. 조정 필요.

- **토르** HP 2300은 과도. 2000으로 하향 또는 ATK를 60으로 낮춤.

### 1-3. 미니언 밸런스

#### 현황 진단

현재 스케일 공식: `let scale = 1 + GS.time/300`

- 5분(300초): 2x HP·ATK
- 10분(600초): 3x HP·ATK  
- 18분(1080초): **4.6x** ← 매우 빠른 성장

이로 인해 후반부 미니언이 영웅급 체력을 가지게 됨.

#### 수정 지침

`Minion` 클래스 constructor 내 scale 계산 수정:

```javascript
// 기존
let scale = 1 + GS.time/300;

// 변경 → 성장 상한선 추가 + 증가폭 완화
let scale = Math.min(1 + GS.time/400, 3.5); // ← 최대 3.5x, 증가속도 완화

this.maxHp = Math.floor(400 * scale);
this.hp    = this.maxHp;
this.atk   = Math.floor(15 * scale);

// 추가: 미니언 골드도 시간에 따라 소폭 증가 (선택)
// onKill에서 미니언 골드를 60G 고정→ 아래로 교체

```

### 1-4. 필드몹(정글) 밸런스

#### 현황 진단

`Monster` 클래스: boss면 HP 5000 / ATK 150, 일반이면 HP 1500 / ATK 40. 모든 일반 몹이 동일 스탯. 5종류 구분이 없음.

#### 수정 지침

`Monster` constructor를 타입별로 분기:

```javascript
constructor(x, y, mtype){
    super(x, y, 'NEUTRAL', 'jungle');
    this.mtype = mtype;
    this.home  = {x, y};

    // 타입별 스탯 분기 (기존 코드 전체를 아래로 교체)
    const statMap = {
        wolf:     { hp: 800,  atk: 28, aspd: 1.1, move: 110, range: 45, radius: 14, respawn: 10 },
        bear:     { hp: 1800, atk: 55, aspd: 0.7, move: 70,  range: 55, radius: 22, respawn: 18 },
        golem:    { hp: 2500, atk: 40, aspd: 0.6, move: 55,  range: 60, radius: 26, respawn: 25 },
        skeleton: { hp: 600,  atk: 35, aspd: 1.3, move: 130, range: 40, radius: 12, respawn: 8  },
        slime:    { hp: 500,  atk: 18, aspd: 1.5, move: 95,  range: 35, radius: 11, respawn: 6  },
    };

    const isBoss = mtype.includes('boss');
    const s = !isBoss && statMap[mtype] ? statMap[mtype] : null;

    if(s) {
        this.maxHp    = s.hp;
        this.atk      = s.atk;
        this.aspd     = s.aspd;
        this.moveSpd  = s.move;
        this.range    = s.range;
        this.radius   = s.radius;
        this._baseRespawn = s.respawn;
    } else {
        // 보스
        this.maxHp   = 5000;
        this.atk     = 150;
        this.aspd    = 0.8;
        this.moveSpd = 80;
        this.range   = 50;
        this.radius  = 40;
        this._baseRespawn = 0; // 보스는 부활 없음
    }
    this.hp = this.maxHp;
    this.respawnTimer = 0;
}
```

onDeath / update 내 `respawnTimer=15` 고정값을 `this._baseRespawn`으로 교체:
```javascript
// Monster.onDeath 내
this.respawnTimer = this._baseRespawn || 15;
```

골드 획득도 타입별로 차등:
```javascript
// Hero.onKill 내 jungle 처리 부분
} else if(target.type==='jungle'){
    const jungleGold = {
        wolf: 50, skeleton: 40, slime: 35,
        bear: 120, golem: 150
    };
    let gAmt = jungleGold[target.mtype] || 100;
    this.gold += gAmt;
    this.gainExp(gAmt * 0.5); // 골드에 비례한 EXP
    if(this.isPlayer) addText(this.x, this.y-40, '+'+gAmt+'G', '#fbbf24', 16);
}
```

## 2. 그르르 거대화 스킬 외형 버그 수정

### 2-1. 버그 원인 분석 (3가지)

#### 버그 1: `grrrGiantTimer` 감소 코드가 없음 ← 가장 치명적

`useSkill()`에서 `this.grrrGiantTimer = cd * 0.66`으로 설정하지만, 이 값을 감소시키는 코드가 **어디에도 없습니다**.  
결과: 거대화 효과가 종료되지 않고 영구 지속 (AI 캐릭터 한정으로 그나마 작동하는 것처럼 보이는 이유는 `handleAI()` 내 체크 조건이 항상 참이 되기 때문).

#### 버그 2: 외형 크기 변경 로직이 `handleAI()` 안에만 있음

```javascript
// handleAI() 내부 (약 250번째 줄 근방)
if(this.grrrGiantTimer > 0) {
    ...
    if(!this.isGiant) {
        this.isGiant = true;
        this.radius *= 1.8; // ← 이 코드가 handleAI()에만 있음!
    }
}
```

플레이어가 그르르를 선택하면 `handlePlayerInput()`이 호출되고 `handleAI()`는 호출되지 않으므로, 플레이어 그르르는 거대화 외형이 **절대 바뀌지 않습니다**.

#### 버그 3: 그르르 그리기 좌표 오류

`drawBlockyHero()` 내 grrr 블록에서 사자 머리와 귀가 `ctx.arc(0, -r*0.6, ...)`처럼 좌표 `0`을 기준으로 그려지지만, 실제 캐릭터 위치는 `x, y` 절대 좌표입니다. ctx는 해당 지점에 translate되지 않은 상태라 머리가 화면 좌상단에 그려집니다.

### 2-2. 수정 지침 (단계별 적용)

#### STEP 1: `Hero.update()` 내 타이머 감소 추가

`Hero.update(dt)` 함수 시작 부분 (isDead 체크 직후)에 추가:

```javascript
// Hero.update(dt) 내 최상단, isDead 리턴 이전에 삽입
if(this.grrrGiantTimer !== undefined && this.grrrGiantTimer > 0) {
    this.grrrGiantTimer -= dt;
}
```

#### STEP 2: isGiant 적용 로직을 `handleAI()`에서 `Hero.update()`로 이동

`handleAI()` 내 아래 블록을 **삭제**:

## 5. 미니맵 HUD 완전 수정

### 5-1. 핵심 버그 원인

**`drawMinimap()` 함수가 절대 실행되지 않습니다.**

```javascript
// 현재 코드 (game.js 최하단 근방)
function drawMinimap(){
    if(!window.mCtx) return;  // ← window.mCtx는 한 번도 설정된 적 없음!
    ...
}
```

`mCtx`는 `let mCtx = mCanvas.getContext('2d')` (지역 변수)로 선언되지만 `window.mCtx`에는 할당되지 않았습니다. 결과적으로 `drawMinimap()`은 항상 첫 줄에서 return합니다.

**추가 문제**: 요구사항은 "라인 상태 + 영웅 위치만 표시"인데 현재 코드는 타워, 넥서스, 보스까지 표시하게 되어 있음.

```javascript
    if (!window.mCtx) return;
    const mc = window.mCtx;

    mc.clearRect(0, 0, W, H);
    mc.fillStyle = '#0f172a';
    mc.fillRect(0, 0, W, H);

    // ── 좌표 변환 헬퍼 ──
    const tx = (wx) => (wx / MAP_SIZE) * W;
    const ty = (wy) => (wy / MAP_SIZE) * H;

    // ────────────────────────────────────────
    // [1] 3라인 도로 표시 (라인 상태)

    mc.fillStyle = `rgba(139,90,43,${laneAlpha})`;

    // 탑 라인 (좌상단 ㄱ자)

    mc.fillRect(tx(200), ty(200), tx(2300), ty(200)); // 가로 (상단 벽)

    mc.fillRect(tx(2500), ty(200), tx(200), ty(2300)); // 세로 (우측 벽)
    mc.fillRect(tx(200), ty(2500), tx(2300), ty(200)); // 가로 (하단 벽)

    // 미드 라인 (대각선)
    mc.save();
    mc.translate(W/2, H/2);
    mc.rotate(-Math.PI/4);
    mc.fillRect(-W*0.58, -H*0.035, W*1.16, H*0.07);
    mc.restore();

    // ────────────────────────────────────────
    // [2] 타워(라인 상태) 표시 — 잔존 여부로 라인 장악 가시화
    // ────────────────────────────────────────
    entities.forEach(e => {
        if(e.isDead) return;
        if(e.type !== 'tower') return; // nexus_turret, nexus 제외

        let col = e.faction === 'BLUE' ? '#3b82f6' : '#ef4444';
        mc.fillStyle = col;
        mc.globalAlpha = 0.6;
        mc.fillRect(tx(e.x) - 2.5, ty(e.y) - 2.5, 5, 5); // 작은 사각형
        mc.globalAlpha = 1;
    });

    // ────────────────────────────────────────
    // [3] 영웅 위치 표시 (핵심)
    // ────────────────────────────────────────
    entities.forEach(e => {
        if(e.type !== 'hero' || e.isDead) return;

        let ex = tx(e.x);
        let ey = ty(e.y);

        if(e === player) {
            // 플레이어: 크고 흰 테두리 + 팀 색
            mc.strokeStyle = '#ffffff';
            mc.lineWidth   = 1.5;
            mc.fillStyle   = e.faction === 'BLUE' ? '#60a5fa' : '#f87171';
            mc.beginPath();

            mc.stroke();

            mc.moveTo(ex, ey - 7);
            mc.lineTo(ex - 3, ey - 13);
            mc.lineTo(ex + 3, ey - 13);
            mc.closePath();
            mc.fill();
        } else {
            // AI 영웅: 팀 색 원
            mc.fillStyle = e.faction === 'BLUE' ? '#3b82f6' : '#ef4444';
            mc.beginPath();
            mc.arc(ex, ey, 3, 0, Math.PI*2);
            mc.fill();
        }
    });

    // ────────────────────────────────────────
    // [4] 카메라 시야 사각형 표시 (현재 화면 범위)
    // ────────────────────────────────────────
    if(player) {

        let visH = (window.innerHeight / camera.zoom) / MAP_SIZE * H;
        let vcx  = tx(camera.x) - visW/2;

    // ────────────────────────────────────────
    // [5] 미니맵 테두리
    // ────────────────────────────────────────

    mc.lineWidth   = 1.5;
    mc.strokeRect(0, 0, W, H);
```

## 6. AI 로직 인간화 개선

### 6-1. 현재 AI의 문제점

1. **상태 전환이 너무 즉각적**: HP 30% 되는 순간 바로 도주 → 너무 기계적.
2. **목표 우선순위 없음**: 가장 가까운 적만 공격, 킬 욕심·백도어 행동 없음.
3. **정글 파밍 AI 없음**: jungle role 영웅도 항상 라인만 섬.
4. **팀 집결 행동 없음**: 한타가 벌어져도 각개 행동.
5. **아이템 구매가 완전 랜덤**: 역할에 맞는 빌드 없음.
6. **감정 표현 없음**: 연속킬 / 피격 / 이동 중 아무 행동 변화 없음.

#### STEP 2: `handleAI(dt)` 전면 개선

`handleAI(dt)` 함수 전체를 아래로 교체합니다:

```javascript
handleAI(dt) {
    if(this.isDead || this.stunTimer > 0) return;

    let myBase  = this.faction === 'BLUE' ? {x:300,  y:2700} : {x:2700, y:300};
    let enyBase = this.faction === 'BLUE' ? {x:2700, y:300}  : {x:300,  y:2700};
    let hpRatio = this.hp / this.maxHp;
    let p       = this.aiPersonality;

    // ── 기본 상점 방문 ──
    if(dist(this, myBase) < 400) {
        this.aiShopTimer -= dt;
        if(this.aiShopTimer <= 0) {
            this.aiShopRoleAware();         // ← 역할 인식 쇼핑 (STEP 3)
            this.aiShopTimer = rand(8, 18);
        }
    }

    // ── 결정 타이머: 인간처럼 0.3~0.8초마다 재평가 ──
    this.aiDecisionTimer -= dt;
    if(this.aiDecisionTimer > 0) {
        // 현재 결정 실행만 함 (재평가 없음)
        this.executeDecision(dt, myBase, enyBase);
        this.autoAttack();
        return;
    }
    this.aiDecisionTimer = rand(0.3, 0.8);  // 다음 재평가 시점

    // ── 근처 정보 수집 ──
    let nearEnemyHeroes = entities.filter(e => e.faction !== this.faction && !e.isDead && e.type==='hero' && dist(this,e) < 600);
    let nearAllyHeroes  = entities.filter(e => e.faction === this.faction  && !e.isDead && e.type==='hero' && e !== this && dist(this,e) < 500);
    let nearEnemyAll    = entities.filter(e => e.faction !== this.faction  && !e.isDead && dist(this,e) < 600);

    // ── 위험도 계산 ──
    let dangerScore = 0;
    nearEnemyHeroes.forEach(e => {
        let killThreaten = (e.atk * e.aspd * 5) > this.hp; // 5초 안에 죽을 수 있으면 위협
        dangerScore += killThreaten ? 2 : 1;
    });
    let underTower = entities.some(t => (t.type==='tower'||t.type==='nexus_turret') && t.faction!==this.faction && !t.isDead && dist(this,t)<t.range+50);

    // ── 도주 결정 ──
    let retreatThreshold = 0.25 + (1 - p.riskTaking) * 0.25; // 낮은 위험감수=높은 임계치
    if(hpRatio < retreatThreshold || (hpRatio < 0.5 && dangerScore >= 3 && underTower)) {
        // 킬 욕심 있고 적 체력이 낮으면 무리해서 추격
        let dyingEnemy = nearEnemyAll.find(e => e.hp/e.maxHp < 0.12 && dist(this,e)<this.range*1.5);
        if(dyingEnemy && p.greediness > 0.6) {
            this.aiCurrentDecision = 'KILL_CHASE';
            this.aiDecisionTarget = dyingEnemy;
        } else {
            this.aiCurrentDecision = 'RETREAT';
        }
    }
    // ── 한타 합류 ──
    else if(nearAllyHeroes.length >= 2 && nearEnemyHeroes.length >= 2 && p.teamPlayer > 0.5) {
        this.aiCurrentDecision = 'TEAMFIGHT';
    }
    // ── 정글 파밍 (정글 역할이거나 정글 선호도 높음) ──
    else if(this.laneRole==='jungle' || (p.junglePref > 0.5 && nearEnemyAll.length === 0)) {
        this.aiCurrentDecision = 'JUNGLE';
    }
    // ── 공격 ──
    else if(nearEnemyAll.length > 0) {
        this.aiCurrentDecision = 'ATTACK';
    }
    // ── 라인 전진 ──
    else {
        // 가끔 쉬기 (Idle)
        if(Math.random() < 0.03) {
            this.aiCurrentDecision = 'IDLE';
            this.aiIdleTimer = rand(0.5, 2.5);
        } else {
            this.aiCurrentDecision = 'LANE';
        }
    }

    this.executeDecision(dt, myBase, enyBase);
    this.autoAttack();
}

executeDecision(dt, myBase, enyBase) {
    let p = this.aiPersonality;

    // 이동 헬퍼
    const moveTo = (tx, ty, spd) => {
        let a = Math.atan2(ty - this.y, tx - this.x);
        // 약간의 방향 노이즈 (인간적 오차)
        a += rand(-0.08, 0.08);
        this.vx = Math.cos(a) * (spd || this.moveSpd);
        this.vy = Math.sin(a) * (spd || this.moveSpd);
        this.facingDir = tx < this.x ? -1 : 1;
    };
    const stopMove = () => { this.vx = 0; this.vy = 0; };

    switch(this.aiCurrentDecision) {

        case 'RETREAT': {
            moveTo(myBase.x, myBase.y);
            if(dist(this, myBase) < 120) stopMove();
            // 도주 중 이모트 (가끔)
            if(!this.emote && Math.random() < 0.005) {
                this.emote = ['😰','🏃','😱'][Math.floor(Math.random()*3)];
                this.emoteTimer = 2;
            }
            break;
        }

        case 'KILL_CHASE': {
            let t2 = this.aiDecisionTarget;
            if(!t2 || t2.isDead) { this.aiCurrentDecision = 'LANE'; break; }
            if(dist(this,t2) > this.range * 0.7) moveTo(t2.x, t2.y);
            else stopMove();
            if(this.heroSkill1Timer <= 0) this.useSkill(1);
            break;
        }

        case 'TEAMFIGHT': {
            let allies = entities.filter(e => e.faction===this.faction && !e.isDead && e.type==='hero' && e!==this);
            let center = allies.reduce((acc,a)=>({x:acc.x+a.x/allies.length, y:acc.y+a.y/allies.length}), {x:0,y:0});
            if(dist(this, center) > 120) moveTo(center.x, center.y);
            else stopMove();
            let tgt = entities.filter(e=>e.faction!==this.faction&&!e.isDead&&dist(this,e)<600)
                               .sort((a,b)=>a.hp-b.hp)[0]; // 가장 체력 낮은 적 우선
            this.aiDecisionTarget = tgt;
            if(tgt && this.heroSkill1Timer <= 0) this.useSkill(1);
            if(tgt && this.heroSkill2Timer <= 0) this.useSkill(2);
            break;
        }

        case 'JUNGLE': {
            let mob = entities.filter(e => e.type==='jungle' && !e.isDead && dist(this,e)<800)
                              .sort((a,b)=>dist(this,a)-dist(this,b))[0];
            if(mob) {
                if(dist(this,mob) > this.range*0.7) moveTo(mob.x, mob.y);
                else stopMove();
            } else {
                moveTo(1500, 1500); // 맵 중앙으로
            }
            break;
        }

        case 'ATTACK': {
            let tgt = entities.filter(e=>e.faction!==this.faction&&!e.isDead&&dist(this,e)<600)
                               .sort((a,b)=>{
                                   // 공격성 높으면 영웅 우선, 낮으면 미니언 우선
                                   if(p.aggression > 0.7) return (b.type==='hero'?-1:1);
                                   return dist(this,a) - dist(this,b);
                               })[0];
            this.aiDecisionTarget = tgt;
            if(!tgt) { this.aiCurrentDecision='LANE'; break; }
            if(dist(this,tgt) > this.range * 0.75) moveTo(tgt.x, tgt.y);
            else stopMove();
            if(this.heroSkill1Timer <= 0 && dist(this,tgt)<400) this.useSkill(1);
            if(this.heroSkill2Timer <= 0 && dist(this,tgt)<400) this.useSkill(2);
            break;
        }

        case 'IDLE': {
            this.aiIdleTimer -= (1/60); // 대략
            stopMove();
            if(this.aiIdleTimer <= 0) this.aiCurrentDecision = 'LANE';
            if(!this.emote && Math.random() < 0.01) {
                this.emote = ['😶','🙄','💭','😴'][Math.floor(Math.random()*4)];
                this.emoteTimer = 2;
            }
            break;
        }

        case 'LANE':
        default: {
            let dest = {x:1500, y:1500};
            if(this.laneRole === 'top') dest = this.faction==='BLUE'?{x:300,y:300}:{x:2700,y:2700};
            if(this.laneRole === 'bot') dest = this.faction==='BLUE'?{x:2700,y:2700}:{x:300,y:300};

            // 목적지에 도달하면 적 기지 방향으로
            if(dist(this, dest) < 300) dest = enyBase;

            // 적 타워 아래면 잠깐 물러섬
            let underTower = entities.some(t=>(t.type==='tower'||t.type==='nexus_turret')&&t.faction!==this.faction&&!t.isDead&&dist(this,t)<t.range+40);
            if(underTower && this.hp/this.maxHp < 0.65) {
                moveTo(myBase.x, myBase.y, this.moveSpd * 0.7);
                break;
            }
            moveTo(dest.x, dest.y);
            break;
        }
    }

    // ── 스탯 재계산 (AI 버프 포함) ──
    let t = HERO_TMPL[this.heroKey];
    if(t) {
        let effAtk = t.atk; let effAspd = t.aspd; let effMove = t.move;
        for(let i=0; i<8; i++) {
            if(!this.inventory[i]) continue;
            let b = BASE_ITEMS.find(x=>x.id===this.inventory[i].id);
            if(!b) continue;
            let lv = 1 + this.inventory[i].upgrade;
            if(b.stat==='atk')  effAtk  += b.val*lv;
            if(b.stat==='aspd') effAspd += b.val*lv;
            if(b.stat==='move') effMove += b.val*lv;
        }
        if(this.soulAtkBonus)     effAtk  += this.soulAtkBonus;
        if(this.underdogBuffTimer > 0) { effAtk *= 1.15; effMove *= 1.15; effAspd *= 1.15; }
        this.atk = effAtk; this.aspd = effAspd; this.moveSpd = effMove;
    }
}
```

#### assists 계산 추가

현재 `Hero.onKill`에서 처치자만 처리됨. 근처 아군에게 어시스트 지급 추가:

```javascript
// Hero.onKill(target) 내 hero 처치 블록 안
if(target.type === 'hero') {
    // 기존 코드 유지 ...

    // ★ 어시스트 처리 (공격에 참여했지만 처치 못한 아군)
    let assistRange = 600;
    entities.forEach(ally => {
        if(ally === this || ally.faction !== this.faction || ally.isDead || ally.type !== 'hero') return;
        if(dist(ally, target) < assistRange) {
            ally.assists = (ally.assists || 0) + 1;
            ally.gold    += 80; // 어시스트 골드
            if(ally.isPlayer) addText(ally.x, ally.y-35, '+80G (어시)', '#a78bfa', 13);
        }
    });
}
```

`buildScoreboard()` 내 KDA 표시 수정:

```javascript
// 기존
const kda = `${h.kills} / ${h.deaths} / 0`; // ← assists 항상 0

// 수정
const kda = `${h.kills} / ${h.deaths} / ${h.assists || 0}`;
```
