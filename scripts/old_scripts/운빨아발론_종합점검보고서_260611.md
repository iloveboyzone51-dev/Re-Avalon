### 1.2 밸런스 문제점 상세 분석

#### 🚨 CRITICAL — 아칸(ARCHON) 완전 파괴 수준

**코드 증거 (autoAttack() 함수):**
```javascript
if(this.heroKey === 'ARCHON') {
    laserEffects.push({...width:15});
    let hitTargets = entities.filter(e=>e.faction!==this.faction && !e.isDead && dist(e, target) <= 80);
    hitTargets.forEach(tgt => { tgt.applyRawDamage(dmg, this); });
}
```

- **모든 기본 공격이 반경 80 광역**: 타겟 주변 80 반경 전체 히트
- **DPS 108 + 광역**: 실효 팀DPS는 표기의 2~3배
- **ASPD 1.8은 전 영웅 최고**: 쿨다운 감소 아이템까지 더하면 스킬 도배 가능
- **스킬1 사이어닉 스톰**: 200반경 광역 + 200반경 0.5초 스턴 + 15발 번개
- **스킬2 마엘스톰**: 180반경 3초 완전 마비 (비헤모스 착용자도 2.1초)

→ **판정: 전 영웅 중 압도적 1위. 솔로 플레이·팀파이트 양쪽 최강.**

#### 🚨 CRITICAL — 크래그(CRAG) 개발자가 직접 명시한 오버파워

```javascript
CRAG: { role_desc:"[탱커 / 근거리 / 오버파워]", hp:4500, atk:55 }
```

- **HP 4,500**: 평균 탱커(2,000~2,800) 대비 1.6~2.2배
- **스킬2 바위 갑옷**: 최대 체력 30% 실드 → 4,500 × 0.3 = 1,350 순간 실드
- **스킬1 대지 강타**: 반경 150 광역 + 1.5초 기절
- **평타 자체가 반경 100 광역** (autoAttack에서 hitTargets 필터 반경 100)
- 방어력 50 + 피해 감소 효과 + 실드 조합 시 실질 효체HP ~8,000 이상

→ **판정: 탱커로서 딜과 제어도 최고. 사실상 픽 강제 수준.**

#### ⚠️ HIGH — 네크로맨서 최저 DPS + 낮은 생존력

- DPS 38 = 전 영웅 최하위
- HP 1,820 (하위권)
- 스킬1 해골 소환: 소환수가 `lifeTimer=15`초 후 소멸, 실질 DPS 기여 낮음
- 스킬2 저주 역병: 이속 감소가 주요 효과지만 딜은 약함
- **아칸 대비 DPS 차이 무려 2.84배**

#### ⚠️ HIGH — 궁수(ARCHER) 사거리 420 = 타워 사거리(360)보다 김

- 타워 사거리 360, 궁수 사거리 **420** → 타워 밖에서 타워 공격 가능
- 블링크(순간이동) + 화살 폭우 콤보로 교전 중 안정적 탈출
- **타워 다이브 패널티 없이 원거리 포킹** 가능 → 라인전 이점 과도

#### ⚠️ MEDIUM — 뱀파이어 기본 흡혈 20% 내장

#### ✅ GOOD — 전반적 독자성

### 1.3 레벨업 성장 시스템 분석

let stats=['atk','hp','move','aspd']; let c=stats[Math.floor(Math.random()*stats.length)];

## 2. 필드몹 종류·수량·경험치 보상 분석 ⭐⭐⭐ (3/5)

### 2.1 현황 데이터

**종류:** 늑대(wolf), 곰(bear), 고렘(golem), 해골전사(skeleton), 슬라임(slime) — 5종  
**총 수량:** 25마리 (맵 전체 랜덤 배치, 400~2600 좌표 범위)  
**리스폰:** 사망 후 **15초** (비보스 기준)

```javascript
// 코드 확인 (Monster 생성자)
this.maxHp = mtype.includes('boss') ? 5000 : 1500;
this.atk   = mtype.includes('boss') ? 150  : 40;
```

### 2.2 핵심 문제: 5종이 모두 동일한 스탯!

```javascript
// wolf, bear, golem, skeleton, slime 모두 동일:
maxHp=1500, atk=40, aspd=0.8, moveSpd=80, range=50
```

**외모는 다르지만 능력치는 완전 동일.** 고렘이 슬라임과 같은 HP·ATK를 가짐. 몬스터별 개성이 전혀 없어 정글 파밍이 단조로움.

### 2.3 보상 구조

| 처치 대상 | 골드 | 경험치 |
|---|---|---|
| 미니언 | 60G | 25XP |
| 정글몹 (비보스) | 100G | 60XP |
| 보스 정글몹 | 400G | 150XP |
| 적 영웅 | 250G | 80XP |
| 타워 파괴 | 전팀 300G + 최종타 200G | - |
| 황금 고블린 | 2,500G | - |

### 2.4 리스폰 타이머 15초는 적절한가?

15초 리스폰은 **매우 빠름** (LoL은 50~120초). 정글러가 계속 같은 캠프를 반복할 수 있어 골드 인플레 발생 가능. 다만 MOBA 특성상 전체 행동 루프가 빠르므로 현 상태 유지도 무방.

### 2.5 개선 방향

- 몬스터 타입별 스탯 차별화 필요 (예: 고렘 고HP저ATK, 늑대 고속저HP)
- 정글몹 배치 완전 랜덤 → 반복 플레이 시 정글 동선 예측 불가 (현재도 준수)
- 정글몹 25마리 대비 맵 크기(3000×3000) 고려 시 밀도는 적절

## 3. 드래곤 시스템 분석 ⭐⭐⭐⭐ (4/5)

### 3.1 시스템 개요

```javascript
// Dragon Spawn Logic
if(GS.time >= 300 && GS.time - GS.lastDragonCheck >= 300) {
    GS.lastDragonCheck = GS.time;
    let scale = Math.floor(GS.time / 300);  // 5분=1배, 10분=2배
    entities.push(new EpicDragon(600, 600, 'red', scale));
    entities.push(new EpicDragon(2400, 2400, 'blue', scale));
}
```

**5분마다 맵 11시(600,600)에 레드 드래곤, 5시(2400,2400)에 블루 드래곤 동시 등장.**

### 3.2 스케일링

| 등장 시점 | 배율 | HP | ATK | 방어 |
|---|---|---|---|---|
| 5분 | 1× | 45,000 | 450 | 75 |
| 10분 | 2× | 90,000 | 900 | 150 |
| 15분 | 3× | 135,000 | 1,350 | 225 |

### 3.3 행동 패턴

1. **지상 페이즈**: 가장 가까운 영웅 추적, 12초마다 500반경 포효(스턴+넉백), 범위 150에서 브레스 2초
2. **비행 페이즈 (HP 50% 이하)**: 8초간 무적 + 반경 500 내 폭격 (atk×1.5)
3. **처치 보상**: 랜덤 2종 스탯 +30% 버프 5분 (공격력, 방어력, 이속, 공속, 최대체력 중 2개)

### 3.4 잘 작동하는 부분

- 2페이즈 전환 연출 (배너 + 링 효과)
- 스케일링이 게임 후반 강제 처치 압박 생성
- 팀 버프 보상이 명확하고 가치 있음

### 3.5 문제점

#### 🔴 재시작 시 드래곤 타이머 리셋 안 됨 — 버그!

```javascript
// startGame()에서 GS.time=0으로 리셋되지만:
if(GS.lastDragonCheck === undefined) GS.lastDragonCheck = 0;
// → 두 번째 게임에서 이전 GS.lastDragonCheck 값 유지
```

**1번 게임 10분에 드래곤 처치 → GS.lastDragonCheck=600**  
**2번 게임 시작: GS.time=0, lastDragonCheck=600**  
**→ GS.time이 300(5분)에 도달해도 300-600=-300, 조건 불만족**  
**→ 2번 게임 9분(=900-600=300)에야 드래곤 등장! 5분이 아니라 9분에.**

#### ⚠️ 기존 드래곤 살아있는 상태에서 새 드래곤 추가 스폰 가능

처치 확인 없이 매 5분마다 신규 드래곤 push. 드래곤이 느려서 처치가 안 됐다면 맵에 최대 4~6마리까지 쌓임.

#### ⚠️ 양 드래곤 위치가 항상 고정 (600,600) (2400,2400)

매 게임 동일한 위치. 전략적 변수 없음.

## 4. 크리처 3종 미등장 원인 분석 — 🔴 버그 확인

> **결론부터: 재시작 시 타이머 미리셋 버그 + 스폰 수량이 1마리뿐**

### 4.1 크리처 스폰 코드 전체 분석

```javascript
// 게임루프 (6분=360초마다)
if(GS.lastCreatureSpawn === undefined) GS.lastCreatureSpawn = 0;
if(GS.time > 0 && GS.time - GS.lastCreatureSpawn >= 360) {
    GS.lastCreatureSpawn = GS.time;
    if(window.spawnCreatures) window.spawnCreatures(1);  // ← count=1 !
}

// spawnCreatures 함수
window.spawnCreatures = function(count) {
    const lanes = ['top', 'mid', 'bot'];
    const types = ['dragon', 'golem', 'beast'];
    showBanner('크리처 군단 소환!', '🐉', true);  // "군단" 이라고 하지만
    for(let i=0; i<count; i++) {  // count=1 → 1마리만!
        let lane = lanes[Math.floor(Math.random() * lanes.length)];
        let ctype = types[Math.floor(Math.random() * types.length)];
        entities.push(new Creature(300, 2700, player.faction, lane, ctype));
    }
};
```

### 4.2 버그 1: GS.lastCreatureSpawn 재시작 시 리셋 안 됨

`startGame()`에서 `GS.time=0`은 리셋되지만 `GS.lastCreatureSpawn`은 undefined 체크로만 초기화됨.

| 상황 | GS.lastCreatureSpawn | GS.time=6분 시 조건 | 결과 |
|---|---|---|---|
| 첫 게임 | 0 (초기) | 360-0=360 ≥ 360 ✅ | 6분에 스폰 |
| 두 번째 게임 | 360 (이전값) | 360-360=0 ≥ 360 ❌ | 미스폰 |
| 두 번째 게임 12분 | 360 | 720-360=360 ✅ | 12분에서야 스폰 |

**→ 재시작할 때마다 6분씩 지연. 실제로는 등장을 못 보는 게임이 많음.**

### 4.3 버그 2: "군단 소환" 이라는데 1마리만 스폰

`spawnCreatures(1)` → 3종 중 1종이 랜덤으로 1마리만 등장.  
배너는 **"크리처 군단 소환!"** 인데 실제론 외로운 1마리.  
시각적 임팩트 없어 플레이어가 인식을 못 함.

### 4.4 버그 3: RED 팀 플레이어 시 위치 오류

```javascript
entities.push(new Creature(300, 2700, player.faction, lane, ctype));
// → 항상 BLUE 본진 위치(300,2700)에 스폰
```

**플레이어가 RED 팀이면 RED 진영 크리처가 BLUE 본진(300,2700)에 스폰됨.**  
RED 미니언의 웨이포인트는 (2700,300)에서 시작하지만, 크리처는 (300,2700)에 있어 반대 방향으로 이동.

### 4.5 버그 4: 크리처 자신의 onDeath 미정의

크리처 사망 시 `applyRawDamage`에서 `this.onDeath(attacker)` 호출 → Creature 클래스에 `onDeath` 없음 → Minion에도 없음 → Entity에도 없음.

→ 사망 처리 시 **onDeath is not a function** 오류 가능 (브라우저 콘솔 확인 필요)

### 4.6 크리처 3종 스탯 (6분 기준, scale=2)

| 종류 | HP | ATK | ASPD | 이속 | 반경 |
|---|---|---|---|---|---|
| 화룡(dragon) | 30,000 | 600 | 0.6 | 120 | 24 |
| 에메랄드 골렘(golem) | 60,000 | 200 | 0.5 | 100 | 30 |
| 황금 마수(beast) | 24,000 | 800 | 1.5 | 250 | 20 |

**만약 제대로 스폰된다면 압도적인 라인 푸시력을 가짐. 타워 HP 13,500 대비 골렘은 HP 60,000으로 타워 4개 분량.**

### 5.1 수호신 코드 구조 문제: 메서드 중복 정의

```javascript
class Guardian extends Entity {
    constructor(...)
    applyRawDamage(...)  // ← 첫 번째 정의 (Dead Code)
    applyStun(...)       // ← 첫 번째 정의 (Dead Code)
    applyRawDamage(...)  // ← 두 번째 정의 (이게 실제 동작)
    applyStun(...)       // ← 두 번째 정의 (이게 실제 동작)
    applySlow(...)
    update(...) {        // ← 첫 번째 update (Dead Code! 절대 실행 안 됨)
        // aggro range 800, 복잡한 로직
    }
    draw(...)
    update(...) {        // ← 두 번째 update (이게 실제 동작)
        // aggro range 400
    }
}
```

**JavaScript 클래스에서 메서드를 두 번 정의하면 마지막이 덮어씀.**  
코드 800줄짜리 로직이 있는 첫 번째 update()는 한 번도 실행되지 않음.

### 5.2 실제 동작 중인 Guardian.update() 분석

```javascript
// 어그로 조건 (활성 update):
let target = entities.find(e => 
    e.faction !== this.faction && !e.isDead && 
    dist(this, e) <= 400 &&          // 수호신으로부터 400 이내
    dist(this.home, e) <= 400        // 본진으로부터 400 이내
);
```

**이중 조건: 수호신 + 본진 모두로부터 400 이내여야 반응.**  
초기에는 같은 위치지만, 수호신이 이동하면 조건이 달라짐.

### 5.3 "존재감 없음"의 이유 5가지

1. **공격사거리 70 + 이동속도 100**: 대부분 영웅(이속 150~185)이 수호신보다 빨라서 영원히 킬링 못 함. 근거리 탱커라도 피하면 그만.

2. **어그로 범위 400**: 본진에서 400 반경 밖은 무반응. 타워들이 먼저 처리하므로 적이 본진 400 내에 도달하는 게 자체가 힘듦. 즉 **수호신이 활성화될 게임 상황 자체가 거의 없음.**

3. **아군 버프/치유 없음**: 본진 귀환 영웅에게 힐이나 버프를 주지 않음. 단순히 "공격만 하는 고정 방어물" 수준.

4. **스케일링만 있고 참여가 없음**: 5분마다 HP·ATK가 1배씩 증가하지만, 영원히 집 근처에서 대기. 10분 HP=100,000, ATK=400이지만 절대 만날 수 없음.

5. **minimap·UI에 표시 없음**: 플레이어가 수호신의 존재를 아예 모를 수 있음.

### 5.4 수호신의 잠재력 (현재 미발휘)

- 50% 피해 감소 내장
- 지진격(ATK×1.5 + 스턴), 신위(ATK×2), 스플래시 평타 3가지 공격 패턴
- 상당히 정성 들여 만든 비주얼 (왕관, 창, 비단 띠, 광배)

**→ 아직 "무대에 서지 못한 배우" 상태.**

## 9. AI 고도화 지능 분석 ⭐⭐⭐ (3/5)

### 9.1 현재 AI 구조 (예상보다 잘 구현됨)

```
상태: RETREAT (35% HP 이하)
      TEAMFIGHT_JOIN (아군 전투 중 + 적 영웅 감지)
      ATTACK (적 감지)
      LANE (기본 라인 이동)
```

**스코어링 기반 타겟 우선순위:**
```javascript
score += (1.0 - e.hp/e.maxHp) * 400;   // 체력 낮은 적 우선
score += (1.0 - d/900) * 300;           // 가까운 적 우선
if (e.type === 'hero') score += 250;    // 영웅 우선
if (enemyTower && noFriendlyMinion) score -= 600; // 타워 다이브 회피
```

**원거리 영웅 kite 동작:**
```javascript
if (HERO_TMPL[this.heroKey].type === 'ranged' && d < this.range * 0.8) {
    // 공격 쿨다운 중일 때 후퇴 이동 (kiting)
}
```

### 9.2 AI의 강점

- 체력 35% 이하 시 자동 귀환 (합리적)
- 아군 전투 합류 판단 (TEAMFIGHT_JOIN)
- 타워 앞 미니언 없으면 다이브 안 함
- 원거리 영웅이 교전 중 뒤로 이동 (kiting 구현)

### 9.3 AI의 약점

#### ⚠️ 쇼핑 완전 무작위

```javascript
aiShopAI(){
    let item=BASE_ITEMS[Math.floor(Math.random()*BASE_ITEMS.length)];
    if(this.gold >= item.cost) this.buyItem(item.id);
}
```

- 크래그(탱커)가 치명타 아이템 살 수 있음
- 네크로맨서가 티아맷(근거리 전용 조건) 살 수 있음
- 롤에서 자기 역할에 맞는 빌드 구성 없음

#### ⚠️ 패시브 스킬도 완전 무작위

```javascript
aiSelectSkill() {
    let pick = available[Math.floor(Math.random()*available.length)];
}
```

AI가 진화 조건 고려 없이 랜덤 픽 → AI의 진화 확률 매우 낮음

#### ⚠️ 드래곤/크리처 등 오브젝트 반응 없음

AI가 드래곤 등장을 감지하고 팀으로 몰리거나 오브젝트 싸움을 하는 로직 없음. 드래곤 등장 시 AI는 라인에서 계속 싸움.

#### ⚠️ 역할별 포지션 목표가 단순

```javascript
if(this.laneRole === 'top') { tx = 300; ty = 300; }
else if(this.laneRole === 'bot') { tx = 2700; ty = 2700; }
else if(this.laneRole === 'mid') { tx = 1500; ty = 1500; }
```

탑: (300,300), 미드: (1500,1500), 봇: (2700,2700) 세 고정 좌표만 향함. 적 타워가 무너진 뒤 목표가 갱신되긴 하지만, 라인 미니언을 따라가는 게 아닌 좌표 추적 방식이어서 주동선 이탈 빈번.

### 9.4 실제 유저 같은 AI를 위한 고도화 방향

| 개선 항목 | 설명 |
|---|---|
| 역할별 쇼핑 | 영웅 type(melee/ranged)과 role(탱/딜/서포)에 따른 아이템 우선순위 가중치 |
| 패시브 스킬 선택 | 진화 조건 고려 + 역할 시너지 패시브 우선 |
| 오브젝트 반응 | 드래곤 등장 시 근처 영웅들이 드래곤 방향으로 이동 의사결정 |
| 텔레포트/귀환 | 극저체력 시 '골드를 소비해 귀환 + 회복 후 복귀' 패턴 |
| 갱킹 시도 | 정글 영웅이 인접 라인 교전 감지 시 합류 |
| 워딩 개념 | 정글 경로 예측 이동 |

### 🟢 STEP 6 — AI 쇼핑 역할 기반 개선

**작업 파일:** `game.js`  
**예상 소요:** 중규모 (1~2시간)

Hero 클래스의 `aiShopAI()` 메서드를 다음으로 교체하세요:

```javascript
aiShopAI(){
    let t = HERO_TMPL[this.heroKey];
    let isMelee = t.type === 'melee';
    let highHp = this.baseMaxHp >= 2500;
    
    // 역할별 아이템 우선순위 가중치
    let weights = {};
    BASE_ITEMS.forEach(i => weights[i.id] = 1);
    
    if(highHp) { weights['warmog'] = 5; weights['hp'] = 4; weights['behemoth_armor'] = 3; }
    if(isMelee) { weights['atk'] = 4; weights['aspd'] = 3; weights['tiamat'] = 2; }
    else        { weights['atk'] = 3; weights['aspd'] = 4; weights['crit'] = 3; }
    if(t.lifeSteal) weights['vamp'] = 4;
    if(this.heroKey === 'NECROMANCER') weights['mage_staff'] = 5;
    if(this.heroKey === 'ARIEL' || this.heroKey === 'DARKPRIEST') {
        weights['legion_shield'] = 5; weights['mage_staff'] = 4;
    }
    
    // 가중치 기반 랜덤 선택
    let pool = [];
    BASE_ITEMS.forEach(i => {
        let w = weights[i.id] || 1;
        for(let j=0; j<w; j++) pool.push(i);
    });
    let item = pool[Math.floor(Math.random() * pool.length)];
    if(this.gold >= item.cost) this.buyItem(item.id);
}
```
