# **⚔️ 운빨 아발론 (v1009+ 리팩토링) 코드 정밀 감사 및 개선 지침서**

**작성 목적:** 최신 수정한 엔진 코드의 결함 디버깅, 프레임 드랍(렉)의 근본적 해결, 스킬 레이어 누수 방지 검증 및 아이템/로그라이크 밸런스 안정화 수식 제안

## **1\. 코드 정상화 여부 및 핵심 버그 검증**

유저가 직접 진행한 대규모 리팩토링은 레이어 누적을 줄이는 데 크게 기여했으나, 몇 가지 치명적인 '동적 상태 누적 버그'와 '메모리 누수'가 새로 발견되었습니다. 아래 항목들을 즉시 점검하여 수정해야 합니다.

### **1.1. \[치명적 버그\] applyStats() 내 스탯 무한 누적 (Vamp, Reflect 등)**

* **원인 분석:** 영웅이 아이템을 사거나 레벨업할 때마다 applyStats()가 호출되어 능력치를 처음부터 재계산합니다. 그러나 공격력(atk), 체력(maxHp) 등은 함수 시작 시 baseAtk, baseMaxHp로 초기화되는 반면, **피흡(vamp), 반사율(reflectRate), 추가 방어력 등은 초기화 코드가 누락**되어 있습니다.  
  이로 인해 **상점을 열고 아이템을 사고팔 때마다 피흡율과 반사율이 10%, 20%, 30%... 무한대로 더해져 상점 거래 후 영웅이 무적이 되거나 평타 한 대에 체력이 풀로 차는 치명적인 현상**이 발생합니다.  
* **해결 작업:** applyStats() 함수 시작 부분에 누적형 특수 스탯들의 초기화 코드를 반드시 주입해야 합니다.

// \[기존 코드 \- game.js \- applyStats() 시작 부분\]  
Hero.prototype.applyStats \= function() {  
    this.maxHp \= this.baseMaxHp;  
    this.atk \= this.baseAtk;  
    this.aspd \= this.baseAspd;  
    this.moveSpd \= this.baseMoveSpd;  
    this.range \= this.baseRange;  
    // ...일부 초기화 누락

// \[개선 코드 \- 누적형 특수 스탯 초기화 추가\]  
Hero.prototype.applyStats \= function() {  
    this.maxHp \= this.baseMaxHp;  
    this.atk \= this.baseAtk;  
    this.aspd \= this.baseAspd;  
    this.moveSpd \= this.baseMoveSpd;  
    this.range \= this.baseRange;  
      
    // 무한 누적 방지를 위한 특수 스탯 강제 초기화  
    this.vamp \= 0;  
    this.reflectRate \= 0;  
    this.defense \= 0;  
    this.critRate \= 0.05; // 기본 치명타율 5%로 리셋  
      
    // 이후 아이템 장착 루프를 돌며 스탯 재가산 진행

### **1.2. \[치명적 버그\] 수호천사의 은총 gaTimer 감쇄 누락으로 인한 무한 부활**

* **원인 분석:** 수호천사 발동 시 gaTimer \= 60.0;으로 쿨타임이 세팅되지만, game.js의 프레임 업데이트 루프(update) 내에서 **gaTimer를 매 프레임 깎아주는(this.gaTimer \-= dt) 논리가 누락**되어 있습니다. 이 때문에 한 번 수호천사가 발동하고 나면 다시는 쿨타임이 돌지 않아 두 번 다시 부활하지 못하는 먹통 상태가 됩니다.  
* **해결 작업:** Hero.prototype.update(dt)의 타이머 감소 영역에 gaTimer 감소 코드를 추가합니다.

// \[개선 코드 \- game.js \- Hero.prototype.update 내 타이머 처리 영역\]  
if (this.gaTimer \> 0\) {  
    this.gaTimer \-= dt;  
}

## **2\. 후반부 렉/프레임 드랍의 근본 원인 분석 및 해결책**

유저가 느낀 "후반으로 갈수록 극심해지는 프레임 드랍"의 진짜 주범은 CPU 연산 과부하와 Canvas 드로잉 레이어의 한계에 있습니다.

### **2.1. 주범 1: 죽은 엔티티(entities)의 잔존으로 인한 ![][image1] 탐색 지옥**

* **현상:** 미니언과 몬스터가 죽으면 isDead \= true가 되지만, **entities 배열에서 영구히 삭제(splice)되지 않고 계속 남아있습니다.**  
* **문제점:** 20분이 지나 미니언 누적 사망자가 800마리를 넘어가면, 살아있는 영웅 10마리가 타겟팅을 잡기 위해 entities 배열을 순회할 때 매 프레임당 800 \* 800 \= 640,000번 이상의 불필요한 거리 계산 연산을 수행하게 됩니다. 이것이 후반부 버벅임의 95%를 차지하는 진짜 원인입니다.  
* **해결 작업:** 사망 애니메이션이 끝난(혹은 죽은 즉시) 엔티티를 entities 배열에서 완전히 제거하는 가비지 컬렉터(GC) 필터 로직을 메인 루프에 주입합니다.

// \[개선 코드 \- game.js \- 전역 update() 함수 내 엔티티 필터링 수식\]  
// 기존: entities.forEach(e \=\> e.update(dt));  
// 변경: 매 프레임마다 죽은 미니언, 몬스터, 투사체를 완전히 쓸어내어 가비지 컬렉션 수행

function updateGame(dt) {  
    // 1\. 엔티티 상태 업데이트 진행  
    entities.forEach(e \=\> e.update(dt));  
      
    // 2\. 가비지 컬렉션: 완전히 소멸된 엔티티 제거 (사망 후 3초 지난 개체 제거 또는 투사체 소멸 시)  
    entities \= entities.filter(e \=\> {  
        if (e.isDead) {  
            // 영웅은 부활해야 하므로 절대 지우면 안 됨\!  
            if (e.type \=== 'hero') return true;  
              
            // 미니언, 몬스터, 이펙트는 죽으면 배열에서 영구 삭제하여 N의 크기를 줄임  
            return false;   
        }  
        return true;  
    });  
}

### **2.2. 주범 2: 무제한적으로 증식하는 파티클(Particles) 레이어**

* **현상:** 스킬 모션을 줄였음에도 불구하고 타격 시 생성되는 파티클 개수가 통제되지 않아 렌더링 병목이 생깁니다.  
* **해결 작업:** 전역 particles 배열의 크기가 150개를 초과할 경우 오래된 파티클부터 즉시 지우는 **하드캡(Hard Cap)** 로직을 spawnParticles에 주입합니다.

// \[개선 코드 \- game.js \- spawnParticles 함수\]  
function spawnParticles(x, y, color, count, speed, duration) {  
    // 렉 방지 하드캡: 현재 파티클이 너무 많으면 스폰 생략하여 프레임 보존  
    if (particles.length \> 150\) {  
        particles.splice(0, count); // 오래된 파티클 즉시 제거  
    }  
      
    for (let i \= 0; i \< count; i++) {  
        particles.push({  
            x: x, y: y,  
            vx: (Math.random() \- 0.5) \* speed,  
            vy: (Math.random() \- 0.5) \* speed,  
            color: color,  
            life: duration,  
            maxLife: duration  
        });  
    }  
}

## **3\. 스킬 모션 최적화 로직 점검 및 피드백**

유저가 진행한 "스킬 모션을 줄여 레이어 중첩 현상을 방지한 리팩토링"은 올바른 방향으로 정밀하게 튜닝되었습니다.

* **동작 평가:** ctx.save()와 ctx.restore()의 쌍이 깨지거나 캔버스 드로잉 컨텍스트 상태가 유실되는 일 없이 깔끔하게 닫혀 있습니다.  
* **추가 제안 (드로잉 부하 감쇄):** 투사체나 장판 스킬을 그릴 때 ctx.shadowBlur나 ctx.shadowColor 같은 **그림자 효과 가동을 최소화**해야 합니다. 브라우저의 2D Canvas 환경에서 실시간 그림자(Blur) 연산은 GPU 가속을 받지 못해 프레임을 갉아먹는 가장 큰 요인입니다. 화려한 외곽선은 단순 ctx.strokeStyle로 대체하는 것을 적극 권장합니다.

## **4\. 신규 장비, 로그라이크 스킬 및 조합 시너지 밸런스 검토**

새로 설계된 아이템들과 시너지 시스템은 기획적으로 매우 훌륭하며 근접 캐릭터의 리턴값을 훌륭하게 보장합니다. 다만, 인게임에서 수치가 꼬이지 않도록 다음과 같이 미세 세부 튜닝을 제안합니다.

### **4.1. 티아맷의 도끼 (tiamat) 광역 데미지 예외 처리**

* **검수 결과:** 티아맷의 광역 피해가 주변 적에게 들어갈 때 e.applyRawDamage(splashDmg, this, false) 형태로 아주 안전하게 호출되었습니다. triggerEffects 인자를 false로 주었기 때문에 **무한 피흡이나 또 다른 광역 데미지 발동 루프에 빠지지 않는 최상의 예외 처리가 완성**되어 있습니다. 안티그래비티 팀은 이 설계를 그대로 유지해야 합니다.

### **4.2. 헤르메스의 비상 장화 (hermes\_boots) 기동성 밸런스 조정**

* **스펙:** 비전투 시 이속 25% 추가 증가.  
* **제언:** nonCombatTimer \> 3.0 조건은 훌륭하나, 아군 미니언을 때리는 파밍 행위 역시 '전투 상태'로 판단될 수 있습니다. 쾌적한 맵 합류를 위해 "적 영웅이나 포탑에 피해를 입지 않은 상태"로 비전투 기준을 완화해 주는 것이 플레이어 관점에서 훨씬 시원시원한 로밍감을 선사합니다.

## **5\. 게임성 극대화를 위한 추가 시스템 제언 (종합 평가)**

'운빨 아발론'이 스팀이나 구글 플레이에 런칭해도 손색없을 정도의 완성도 높은 게임이 되기 위해 반드시 추가되어야 할 개발자 관점의 최종 피드백입니다.

### **5.1. 항시 가동되는 '자동 평타(Auto-Attack) 락온 타겟 라인' 제공**

* **필요성:** 뱀서라이크 스타일의 패시브와 탑뷰 MOBA 조작이 섞여 있어, 내 캐릭터가 현재 어떤 미니언이나 영웅을 타겟팅하여 평타를 치고 있는지 직관적으로 보이지 않는 답답함이 존재합니다.  
* **해안:** 플레이어 영웅이 타겟팅한 대상 아래에 지속적으로 회전하는 작은 붉은색 격자 혹은 타겟 링(Target Ring)을 그려 가시성을 150% 확보합니다.

// \[추천 추가 코드 \- game.js 내 플레이어 전용 타겟 렌더링\]  
if (player && player.target && \!player.target.isDead) {  
    ctx.strokeStyle \= 'rgba(239, 68, 68, 0.6)';  
    ctx.lineWidth \= 2;  
    ctx.beginPath();  
    ctx.arc(player.target.x, player.target.y, player.target.radius \+ 10, 0, Math.PI \* 2);  
    ctx.stroke();  
}

### **5.2. 로그라이크 패시브 스킬 쿨타임 가시성 개편**

* 뱀서라이크식 패시브 무기들이 발동할 때, 화면 상단이나 버프 창에 **스킬별 내부 쿨타임(Cool-down) 게이지**가 아주 작게나마 표시되어야 플레이어가 "아, 내 번개 스킬이 곧 치겠구나\!"를 인지하고 적진으로 들어가는 전략적 주도 플레이가 가능해집니다.

## **6\. 안티그래비티 팀 최종 패치 가이드라인 요약**

안티그래비티 팀은 본 지침서를 수령하는 즉시 아래의 작업 순서대로 핫픽스를 단행하십시오.

1. **Hero.prototype.applyStats** 함수 최상단에 스탯 초기화 가이드(vamp, reflect, defense 등 0으로 리셋)를 주입할 것.  
2. 메인 루프의 updateGame에서 사망한 미니언/몬스터 엔티티들을 filter를 통해 배열에서 완전히 소멸시키는 가비지 컬렉터를 구축하여 렉을 100% 제거할 것.  
3. **spawnParticles** 내부의 파티클 스택 크기 하드캡(150개)을 적용하여 프레임 방어를 확보할 것.  
4. **gaTimer** 감소 공식을 Hero.prototype.update에 적용해 수호천사 60초 쿨타임을 정상 작동시킬 것.
