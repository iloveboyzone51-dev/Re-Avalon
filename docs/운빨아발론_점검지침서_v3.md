### 🔴 CRIT-01: 게임 재시작 시 타이머 플래그 누수 — 두 번째 게임부터 고블린·서든데스·드래곤 미스폰

**원인**: `goblinSpawned`, `suddenDeathTriggered`, `midBossSpawned`, `GS.lastDragonCheck`, `GS.lastUnderdogCheck`가 `startGame()`에서 초기화되지 않음. 두 번째 판부터:
- 황금 고블린이 절대 스폰되지 않음 (`window.goblinSpawned = true` 유지)
- 서든데스가 즉시 발동 (`window.suddenDeathTriggered = false`지만 `GS.time`이 0이므로 18분에만 발동 — 이건 OK. 그러나 `window.suddenDeathTriggered`가 이전 판에 true로 설정되어 두 번째 판에는 발동 안 됨)
- 드래곤 체크가 `GS.time - GS.lastDragonCheck >= 300`인데 `GS.lastDragonCheck`가 이전 판의 높은 값 유지 → 드래곤 스폰 안 됨
- 중간보스가 절대 스폰 안 됨 (midBossSpawned 미초기화)

**수정**: `startGame()` 함수 내 초기화 블록에 추가:

```javascript

window.goblinSpawned = false;
window.suddenDeathTriggered = false;
```

### 🔴 CRIT-03: `Projectile.update()` — 화상/기절이 이중으로 적용됨

**원인**: 투사체가 명중할 때 `applyRawDamage(triggerEffects=true)`가 내부에서 burn/stun을 처리하고, `Projectile.update()`도 동일한 처리를 중복으로 수행:

```javascript
// applyRawDamage() 내 (triggerEffects=true시 이미 처리됨)
if (attacker.burnDmg > 0) { this.burnTicks.push(...); }
if (attacker.stunChance > 0 && Math.random() < attacker.stunChance) this.stunTimer = ...

// Projectile.update()에서 또 처리됨 (중복!)
if(this.attacker.burnDmg>0&&!tgt.isBuilding) tgt.burnTicks.push({...});
if(this.attacker.stunChance>0&&Math.random()<this.attacker.stunChance&&!tgt.isBuilding) tgt.stunTimer=1.0;
```

결과: 원거리 영웅은 기절 확률이 2배, 화상이 2스택씩 쌓임.

**수정**: `Projectile.update()` 내 hitTargets forEach 에서 중복 처리 삭제:

```javascript
// Projectile.update() 내 hitTargets.forEach 블록
hitTargets.forEach(tgt => {
    let dealt = tgt.applyRawDamage(this.dmg, this.attacker);  // ← 내부에서 burn/stun 처리됨
    if(this.attacker && this.attacker.type === 'hero') this.attacker.totalDmg += (dealt || this.dmg);
    if(this.attacker && this.attacker.triggerOnHitPassives) this.attacker.triggerOnHitPassives(tgt);
    if(this.attacker.lifeSteal > 0 && this.attacker.type === 'hero') {
        this.attacker.hp = Math.min(this.attacker.maxHp, this.attacker.hp + this.dmg * this.attacker.lifeSteal);
        playSFX('heal');
    }
    // ★ 아래 두 줄 삭제 (이미 applyRawDamage 내에서 처리됨)
    // if(this.attacker.burnDmg>0...) tgt.burnTicks.push(...)   ← 삭제
    // if(this.attacker.stunChance>0...) tgt.stunTimer=1.0;     ← 삭제
    
    // 파티클만 유지
    let hitColor = this.isCrit?'#ff6b35':'#fbbf24';
    ...
    spawnParticles(tgt.x, tgt.y-tgt.radius*0.5, hitColor, 10, 120, 0.3);
});
```

### 🔴 CRIT-04: `BARBARIAN` 휠윈드 중 플레이어 입력이 vx/vy를 즉시 초기화

**원인**: 스킬2 발동 시 `this.vx = Math.cos(a)*600; this.vy = Math.sin(a)*600;`을 설정하지만, 같은 프레임의 `handlePlayerInput()`이 맨 처음에 `this.vx=0; this.vy=0;`을 실행해 돌진 속도를 지워버림. AI도 마찬가지로 `handleAI()`가 vx/vy를 재설정함.

**수정**: `Hero.update()` 내 `handlePlayerInput`/`handleAI` 호출 이전에 휠윈드 이동 강제 적용:

```javascript
// Hero.update() 내 calculateDynamicStats() 호출 이후에 삽입
if(this.whirlwindTimer > 0 && this.whirlwindDashVx !== undefined) {
    // 돌진 중: 외부 입력 무시하고 돌진 방향 강제 적용
    this.vx = this.whirlwindDashVx * (this.whirlwindTimer / 3.0); // 서서히 감속
    this.vy = this.whirlwindDashVy * (this.whirlwindTimer / 3.0);
    // handlePlayerInput/handleAI를 건너뜀
    super.update(dt);
    if(GS.status === 'PLAYING') {
        this.autoAttack();
        this.autoUseHeroSkills();
    }
    this.updatePassives(dt);
    return; // ★ 조기 리턴으로 handleAI/handlePlayerInput 스킵
}
```

그리고 `useSkill()` 내 BARBARIAN 휠윈드 블록에서:
```javascript
} else { // 휠윈드
    this.whirlwindTimer = 3.0;
    this.whirlwindDmg = skillDmg * 0.8;
    
    let tg = t;
    if(!tg) tg = entities.filter(e=>e.type==='nexus' && e.faction!==this.faction)[0];
    if(tg) {
        let a = Math.atan2(tg.y - this.y, tg.x - this.x);
        // ★ 돌진 방향 저장 (vx/vy 직접 설정 대신)
        this.whirlwindDashVx = Math.cos(a) * 500;
        this.whirlwindDashVy = Math.sin(a) * 500;
    } else {
        this.whirlwindDashVx = this.facingDir * 500;
        this.whirlwindDashVy = 0;
    }
    ...
}
```

`whirlwindTimer`가 0이 되면 자동으로 whirlwindDashVx/Vy 초기화:
```javascript
// Hero.update() 내 whirlwindTimer 처리 블록에 추가
if(this.whirlwindTimer > 0) {
```

### 🔴 CRIT-05: `ARCHON` 평타가 자동으로 AoE 스플래시 — 공속 2.34로 게임 내 최강 존재

**원인**: ARCHON 평타는 일반 투사체 대신 `laserEffects` + 즉발 AoE 처리를 사용:
```javascript
if(this.heroKey === 'ARCHON') {
    let hitTargets = entities.filter(e=>e.faction!==this.faction && !e.isDead && dist(e, target) <= 80);
    hitTargets.forEach(tgt => { tgt.applyRawDamage(dmg, this); });
```
반경 80 범위 내 전체 대상에게 즉발 피해. 기본 공속 1.8 * 1.3 = **2.34** → 초당 2.34번 AoE = 게임에서 압도적 최강.

**수정**: ARCHON 평타를 단일 타겟 레이저 방식으로 변경, AoE는 passive/스킬로만:

```javascript
// autoAttack() 내 ARCHON 블록 교체
if(this.heroKey === 'ARCHON') {
    this.attackAnimTimer = 0.2;
    // ★ 단일 타겟 레이저 (스플래시 제거)
    laserEffects.push({
        x1: this.x, y1: this.y - this.radius,
        x2: target.x, y2: target.y - target.radius,
        color: '#ffffff', life: 0.1, maxLife: 0.1, width: 8  // 얇게
    });
    let dealt = target.applyRawDamage(dmg, this);
    this.totalDmg += dealt;
    this.triggerOnHitPassives(target);
    if(this.lifeSteal > 0) this.hp = Math.min(this.maxHp, this.hp + dealt * this.lifeSteal);
    if(this.isPlayer) playSFX('shoot');
}
```

그리고 ARCHON 기본 공속도 조정:
```javascript
// HERO_TMPL
ARCHON: { ..., aspd: 1.4, ... }  // ← 1.8 → 1.4 (×1.3 = 실제 1.82, 보다 적정)
```

### 🟠 BUG-10: `EpicDragon` 비행 단계에서 `invincibleTimer=0.5`를 매 프레임 재설정 → Entity.update가 vx/vy 초기화

비행 중 아래 코드가 매 프레임 실행됨:
```javascript
if(this.flightTimer > 0) {
    this.flightTimer -= dt;
    this.invincibleTimer = 0.5;  // ← 매 프레임 재설정
    // 폭격 코드...
    return;
}
```
`Entity.update()`는 `invincibleTimer > 0`이면 `vx=0; vy=0; return;` 처리. `EpicDragon.update()`에서 `super.update(dt)` 호출이 먼저이므로 EpicDragon은 비행 중 이동이 불가. 시각적으로 드래곤이 비행 중 제자리에 멈춤.

**수정**: 비행 중 무적은 별도 플래그로 관리, invincibleTimer 매 프레임 재설정 제거:

```javascript
if(this.flightTimer > 0) {
    this.flightTimer -= dt;
    // ★ invincibleTimer 매 프레임 재설정 제거 → 비행 시작 시 한 번만 설정
    // this.invincibleTimer = 0.5;  ← 삭제
    
    // 폭격 패턴 (위치 위에서 랜덤 이동)
    // 비행 중 느리게 이동
    let cx = 1500, cy = 1500;  // 맵 중앙
    let da = Math.atan2(cy - this.y, cx - this.x);
    this.vx = Math.cos(da) * 60; this.vy = Math.sin(da) * 60;
    
    if(Math.random() < 0.1) { ... 폭격 ... }
    return;
}
```

비행 시작 시 한 번만 무적 설정:
```javascript
if(this.hp < this.maxHp * 0.5 && this.phase === 1) {
    this.phase = 2;
    this.flightTimer = 8.0;
    this.invincibleTimer = 8.0;  // ★ 8초 무적 한 번만 설정
    showBanner(...);
    spawnRing(...);
}
```

#### 3) 정글 파밍 AI 추가 (laneRole === 'jungle' 영웅)

현재 `laneRole === 'jungle'` 영웅도 `LANE` 상태로 처리되어 적 기지로 달려감. 정글 몬스터를 파밍하는 AI 추가:

```javascript
// handleAI() switch 내 LANE case 앞에 추가
case 'JUNGLE': {
    let jungles = entities.filter(e => e.type === 'jungle' && !e.isDead && dist(this, e) < 800);
    let nearestJungle = jungles.sort((a,b) => dist(this,a) - dist(this,b))[0];
    if(nearestJungle) {
        moveToTarget(nearestJungle, this.range * 0.8, this.range * 0.4);
        if(this.heroSkill1Timer <= 0 && dist(this, nearestJungle) < 300) this.useSkill(1);
    } else {
        // 정글 없으면 가까운 드래곤 어시스트
        let dragon = entities.find(e => e.type === 'epic_dragon' && !e.isDead);
        if(dragon) moveToTarget(dragon, 400, 200);
        else this.aiState = 'LANE';
    }
    break;
}
```

그리고 `handleAI()` AI 상태 결정 로직에 추가:
```javascript
// laneRole이 jungle이고 체력 50% 이상이면 정글 파밍
else if(this.laneRole === 'jungle' && hpRatio > 0.5 && nearEnemies.length === 0) {
    this.aiState = 'JUNGLE';
}
```

#### 4) 영웅별 아이템 추천 시스템 (AI 개선)

현재 AI는 완전 랜덤 쇼핑. 역할별 선호 아이템을 갖도록:

```javascript
// Hero.aiShopAI() 함수 교체
aiShopAI() {
    const rolePref = {
        BERSERKER: ['atk', 'stun', 'hp', 'bork'],
        ARCHON:    ['aspd', 'mage_staff', 'crit', 'atk'],
        BARBARIAN: ['hp', 'atk', 'warmog', 'bork'],
        ARCHER:    ['crit', 'aspd', 'atk', 'move'],
        NECROMANCER: ['mage_staff', 'atk', 'hp', 'warmog'],
        grrr:      ['hp', 'atk', 'warmog', 'reflect'],
        VAMPIRE:   ['vamp', 'atk', 'hp', 'aspd'],
        THOR:      ['atk', 'hp', 'stun', 'bork'],
        ICEBORN:   ['mage_staff', 'hp', 'atk', 'aspd'],
        JOKER:     ['crit', 'aspd', 'atk', 'move'],
        DARKPRIEST:['mage_staff', 'atk', 'hp', 'vamp'],
    };
    let prefs = rolePref[this.heroKey] || [];
    
    // 선호 아이템 중 살 수 있는 것 먼저
    for(let pid of prefs) {
        let item = BASE_ITEMS.find(i => i.id === pid);
        if(item && this.gold >= item.cost) {
            this.buyItem(item.id);
            return;
        }
    }
    
    // 없으면 랜덤
    let affordable = BASE_ITEMS.filter(i => this.gold >= i.cost);
    if(affordable.length > 0) {
        this.buyItem(affordable[Math.floor(Math.random()*affordable.length)].id);
    }
}
```
