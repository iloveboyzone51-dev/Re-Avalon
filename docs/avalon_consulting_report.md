# 🏰 운빨 아발론 (Lucky Avalon) — 종합 게임 컨설팅 보고서

> **보고서 유형:** 게임 밸런스 & 시스템 전략 컨설팅  
> **분석 버전:** v1.0.14 (game.js v4.0 엔진 기준)  
> **분석 범위:** 영웅 밸런스 · 아이템 · 성장 시스템 · 경제 · 필드 · AI · 스킬  
> **제작사에 대한 기본 평가:** 브라우저 기반 5:5 MOBA로서 완성도 높은 시도. 다만 지금 단계에서 냉정한 메스가 필요한 구조적 문제들이 다수 존재함.

## Executive Summary (경영진 요약)

운빨 아발론은 **뱀서라이크 성장**과 **MOBA**를 결합한 독창적 컨셉을 갖고 있으며, 단일 HTML 파일 게임으로는 놀라운 완성도를 보여준다. 그러나 현재 상태는 **"아이디어가 기술을 앞서나가고, 밸런스가 콘텐츠를 따라가지 못하는"** 전형적인 인디 개발 과부하 상태다.

핵심 문제는 다음 세 가지로 압축된다:

1. **원거리 영웅의 타워 관통 사거리** — 게임의 핵심 긴장감(타워 수호)을 붕괴시키는 가장 심각한 설계 결함
2. **골드 경제 과잉** — 황금 고블린 2,500G를 포함한 다층적 골드 수급이 아이템 선택의 전략성을 희석시킴
3. **AI의 아이템 구매 무작위성** — AI가 영웅 타입 무관하게 랜덤 아이템을 구매해 AI 영웅들이 잠재력의 절반도 발휘 못함

이 세 가지만 고쳐도 게임 경험은 한 단계 도약한다.

### 1-2. 타워와의 사거리 비교 — 가장 심각한 문제

코드를 직접 확인한 결과, 타워(tower)의 사거리는 **360**, 수호 타워(nexus_turret)의 사거리는 **350**이다.

```
타워 사거리: 360 / 수호 타워: 350
실비아:    450 → 타워 밖 90 유닛에서 안전 공격 가능 ❌❌
궁수:      420 → 타워 밖 60 유닛에서 안전 공격 가능 ❌❌
아리엘:    390 → 타워 밖 30 유닛에서 안전 공격 가능 ❌
암흑사제:  380 → 타워 밖 20 유닛에서 안전 공격 가능 ❌
제피르:    380 → 타워 밖 20 유닛에서 안전 공격 가능 ❌
조커:      360 → 타워와 동일 사거리 (경계선 위험) △
저격수 렌즈 장착 시 사거리 +30(1강 기준) ~ +45 추가 → 상황 더 악화
```

**개선 방향 (제작사 의견 반영):**  
기본 사거리는 반드시 타워 사거리(360) 미만으로 설정하고, 사거리 아이템 최대 강화 시에도 타워 사거리와 동률(360)을 초과하지 않도록 캡핑한다.

**구체적 수치 제안:**

|------|------|------|
| 실비아 | 450 | 310 |

| 암흑사제 | 380 | 290 |
| 제피르 | 380 | 285 |

저격수 렌즈 아이템의 `val`도 30→15 또는 최대 강화 시 사거리 총합이 360을 초과하지 못하도록 **range 상한선(rangeCap = 355)** 로직 추가를 권장한다.

### 1-3. 실비아(실비아) — 단독 OP 판정

실비아는 현재 **게임 내 최대 ATK(90) + 최고 크리율(20%) + 최장 사거리(450)**를 동시에 보유한다. 이 세 가지가 겹치면 특정 빌드에서는 ASPD가 낮다는 페널티가 의미 없어진다. 특히 관통하는 섬광(1초 채널 후 트루데미지 관통 레이저)과 결합하면 라인전 자체가 성립하지 않는다.

**권장 조정:**  
- ATK 90 → 72 (실제 DPS는 ASPD와 크리율로 차별화 유지)  
- critChance 0.20 → 0.15  
- 사거리 310으로 하향 (상기 표 참조)

### 1-4. 그르르(grrr) — 과잉 스케일링 의심

그르르는 draw 함수에서 `ctx.scale(1.3, 1.3)` 처리로 시각적으로 30% 더 크게 표시된다. 거대화 스킬 발동 시 isGiant 플래그로 `maxHp *= 1.5`가 추가된다. 70 ATK + 폭주 버프 + 체력 1.5배는 스킬 쿨(24s)이 길더라도 스노볼 시 억제가 불가능한 수준이 될 수 있다. 포효(2초 광역 기절, 쿨 18s) 콤보 때 아군 지원이 없으면 탈출 수단이 없다.

**권장 조정:** 거대화 시 maxHp 보정 1.5 → 1.3, 포효 기절 2초 → 1.5초

### 1-5. 아칸 — 숨겨진 약자

range 150으로 "원거리"로 분류되어 ranged 아이템(저격수 렌즈, 루난의 허리케인 등)을 구매 가능하지만 실질적으로는 150 사거리는 근접 영웅(80~120)과 큰 차이가 없다. 그러나 HP 1,820에 이동속도 105(=150×0.7)로 팀에서 가장 느리다. 마엘스톰(완전 마비) 스킬의 가치는 크지만 접근하기 전에 처치당하는 상황이 빈발할 것이다.

**권장 조정:** HP 1,820 → 2,100 또는 MOVE 150 → 165

## 제2장. 아이템 밸런스 · 종류 · 가짓수 분석

### 2-1. 현황

- BASE_ITEMS: 24종 (근접 6, 원거리 6, 마법 6, 공용 6)  
- EVOLUTION_ITEMS: 14종  
- 인벤토리 최대 10슬롯, 강화 최대 +9 (강화 성공률: 100%→100%→100%→60%→50%→40%→30%→20%→10%)

### 2-2. 진단: 타입 분류 불일치

현재 아칸은 `type:"ranged"`이므로 근접 전용(flame_armor, tiamat 등)을 살 수 없다. 그러나 사거리 150은 근접 플레이를 강요한다. 반대로 제로스(range 120)는 `type:"melee"`이지만 실질적 브루저+암살자라 공용 아이템만으로 빌드를 구성해야 한다. **영웅 타입과 아이템 타입 분류가 영웅 역할군과 일치하지 않는 케이스가 다수 존재한다.**

### 2-3. 진단: 구매 단가 불균형

```
수호천사의 은총 (guardian_angel): 2,800G — 사망 시 부활
지옥의 망치 (hell_hammer):        2,000G — 강화 불가
죽음의 모자 (rabadon):            1,800G — 스킬 딜 30%↑
괴수의 뼈갑옷 (behemoth_armor):   1,800G
```

수호천사의 은총 2,800G는 지나치게 고가이면서도 효과 자체는 치트에 가깝다. 골드가 이미 넘쳐흐르는 경제 구조에서 8~10분이면 구매 가능해진다. "60초 쿨"이라는 제한이 있지만 부활 후 회복 아이템과 패시브 스킬 시너지로 사실상 무력화된다.

**권장 조정:** guardian_angel 3,500G로 상향, 부활 쿨다운 90초로 연장

### 2-4. 진단: 아이템 수의 적절성

24개 BASE_ITEMS는 영웅 15종을 지원하기에 부족하지 않다. 다만 **마법사 전용 6종 중 일부가 피흡/이속 패시브와 결합해 유일한 빌드 트리만 형성**되는 경향이 있다(CDR → 죽음의 모자 → 공허의 지팡이가 사실상 마법사 필수 코어). 공용 아이템에 순수 AP 증가 아이템이 없어 서포터/마법사 혼성 영웅(아리엘, 암흑사제)의 빌드 다양성이 낮다.

**신규 아이템 제안 (선택 사항):**
- `공허의 수정` (공용, 700G): 마법/물리 방어 관통 10% 증가 — 공용 마법 지원선
- `생명의 씨앗` (공용, 800G): 비전투 회복 속도 2배 — 초반 라인전 지속력 제공

### 2-5. 진단: 강화 시스템 수익성 문제

`ENHANCE_RATES = [1,1,1,0.6,0.5,0.4,0.3,0.2,0.1]`

3강까지는 100% 성공. 그러나 +4~+9 구간이 대폭 하락한다. 진화 조건이 "7강 달성"이므로 +4~+7 구간 성공률(60%→50%→40%→30%)에서 아이템 원가의 2~4배를 낭비해야 진화 가능하다. 이는 **전략적 강화 결정이 아닌 순수 골드 소진 도박**이 된다.

**권장 조정:**
```
현재: [1, 1, 1, 0.6, 0.5, 0.4, 0.3, 0.2, 0.1]
수정: [1, 1, 1, 0.7, 0.6, 0.5, 0.4, 0.3, 0.2]
```
또는 진화 조건을 7강 → 5강으로 완화하되 추가 조건(패시브 스킬 Max) 유지.

## 제3장. 성장 시스템 분석

### 3-1. 현황 정확 정리 (제작사 코멘트 반영)

> "레벨업 시 스킬이 강제로 들어온다"는 분석은 오독이었다. 실제로는 레벨업 시 3장의 카드가 제시되고 플레이어가 하나를 선택한다.

코드를 재확인한 결과:
- 레벨업 시 **운빨 스탯(atk/hp/move/aspd 중 무작위 1개)** 이 자동 적용된다 — 이 부분이 강제
- 이후 **패시브 스킬 3선택지**가 팝업 — 이 부분이 플레이어 선택

즉 구조는: `레벨업 → 무작위 스탯 자동 부여 → 패시브 스킬 3선택 팝업`

### 3-2. 진단: 무작위 스탯의 역할

레벨당 자동 스탯 부여:
```
atk:  +6
hp:   +60 (즉시 회복 포함)
move: +2.5
aspd: +0.12
```

이 수치는 히어로별 기본값 대비 **영향이 균등하지 않다**. 예를 들어 이미 빠른 제로스(aspd 1.82)에게 aspd +0.12는 미미하지만, 느린 그르르(aspd 1.17)에게는 상대적으로 강한 버프다. 마찬가지로 atk +6은 실비아(90 ATK)에게는 6.7%지만 아리엘(45 ATK)에게는 13.3% 증가다.

**"운빨" 콘셉트를 살리면서도 허탈감을 줄이는 방법:**  
스탯 부여 시 영웅 타입별 가중치를 추가하되, 진짜 운빨 요소(±50% 변동폭)를 함께 적용해 "운이 좋으면 크게, 나쁘면 작게"라는 흥미 포인트를 강화.

```javascript
// 제안 예시
let baseGain = {atk:6, hp:60, move:2.5, aspd:0.12};
let multiplier = 0.5 + Math.random() * 1.5; // 0.5x ~ 2.0x 변동
// 예: atk 3~12 범위, hp 30~120 범위
```

### 3-3. 진단: 패시브 스킬 선택지 다양성 문제

총 PASSIVE_SKILLS: 13종 (근접 6, 원거리 3, 공용 4)  
근접 영웅은 10가지(근접 6 + 공용 4), 원거리 영웅은 7가지(원거리 3 + 공용 4)에서만 선택 가능.

**원거리 영웅의 선택 풀이 절반 이하**다. 후반에 스킬이 쌓일수록 원거리 영웅은 3종의 원거리 전용 스킬(빙결의 손길, 질풍, 그림자 일격)을 필연적으로 다 찍거나 공용으로만 채워진다. **빌드 다양성 부재**.

**권장 신규 원거리 패시브 스킬 (2~3종 추가):**
- `맹독 화살` (maxLv:3): 타격 시 확률로 독 스택 부여, 초당 지속 피해
- `폭발 화살` (maxLv:4): 원거리 타격 시 소형 범위 폭발
- `집중 사격` (maxLv:3): 동일 대상 연속 공격 시 피해 누적 증가

### 3-4. 진단: 진화 시스템의 진입 장벽

진화 조건: `아이템 7강 달성 + 해당 패시브 스킬 Max`

문제는 진화 아이템의 존재를 플레이어가 인지하기 어렵다는 것이다. 도감(🌟) 버튼을 직접 열어야 보이며, 인게임 중 "이 패시브를 올리면 어떤 진화에 가까워진다"는 힌트가 카드 선택 UI에 작게 표시되긴 하지만 존재감이 약하다.

또한 **진화 14종 중 다수가 근접 전용 아이템(tiamat, absolute_armor 등) 조합**이라 원거리/마법 영웅의 진화 경로가 상대적으로 협소하다.

18분 게임 기준 패시브 골드만 **1,080G**. 미니언을 꾸준히 파밍하면 18분간 약 **6,000~8,000G** 수급 가능 (라인당 9초에 소환되므로 분당 약 20마리 × 60G × 3라인 × 50% 먹었다고 가정). 타워 파괴 골드, 영웅 킬 포함 시 게임 중반 이후 골드 상한에 사실상 도달한다.

### 4-2. 황금 고블린 문제

제작사 스스로 "이미 골드가 흘러넘치는데 황금 고블린은 필요없다"고 판단하고 있으며 이 판단은 **완전히 옳다.**

8분에 단 한 번 등장해 2,500G를 주는 황금 고블린은 가장 비싼 아이템(수호천사 2,800G)을 거의 단숨에 살 수 있는 금액이다. 기존 골드 경제가 이미 풍족한 상황에서 이 이벤트는 **경제 파괴 요소**에 가깝다.

**권장 사항:** 황금 고블린 삭제. 대신 해당 타임슬롯(8분)에 **전략적 선택지를 부여하는 이벤트**로 대체 가능:
- `비밀 상인 등장` (8분): 통상 미판매 특수 아이템 2~3종 한정 판매 (정가 대비 10% 할인)
- `정보전 크리스탈` (8분): 처치한 팀에게 적 미니맵 2분간 완전 공개

### 4-3. 팀 금고(TEAM VAULT) 메커니즘 문제

```javascript
// 현재 코드
let blueCount = Math.floor(window.TEAM_VAULT.gold / 450);
if(blueCount > 0) {
    window.TEAM_VAULT.gold -= blueCount * 450;
    // BLUE 팀 크리처 소환
    for(let i=0; i<blueCount; i++) { ... }
}
// RED 팀도 blueCount만큼 동일하게 소환 ← 버그!
if(blueCount > 0) {
    for(let i=0; i<blueCount; i++) {
        entities.push(new Creature(2700, 300, 'RED', lane, ctype));
    }
}
```

**버그 발견:** BLUE 팀이 금고를 축적해 크리처를 소환할 때, RED 팀도 같은 수의 크리처를 무료로 소환받는다. 이는 팀 금고 시스템의 전략적 의미를 완전히 무력화한다. RED 팀 AI는 별도 금고 로직이 없어 이 혜택을 공짜로 받는 구조다.

**수정 방법:** RED 팀 금고(`TEAM_VAULT_RED`) 분리 운영 또는 RED 팀 크리처 소환 블록을 독립적으로 구현.

## 제5장. 필드몹 및 미니언 분석

### 5-1. 미니언 시스템

- 소환 주기: 9초 (3라인 × 양팀)  
- 스케일: `min(1 + time/400, 3.5)` → 18분에 최대 3.7배 (캡 3.5배)  
- 기본 스탯: HP 400, ATK 15, ASPD 1.0, MOVE 120, RANGE 30

**9초 소환 주기는 너무 빠르다.** 미니언 웨이브가 거의 끊기지 않고 푸시되어 타워가 단독으로 버티기 어렵고, 영웅이 미니언 처치에 집중하지 않아도 라인이 자동으로 밀린다. 전략적 라인 관리의 의미가 희석된다.

**권장:** 소환 주기 9초 → 14초 (원래 18초에서 단축된 것인데, 9초는 너무 공격적)

### 5-2. 정글 몬스터 분석

정글 몬스터는 `type: 'jungle'`로 구분되며 100G/60XP를 드랍한다. 중간보스(MID_BOSS_TIMES: 5, 10, 15분)는 400G/150XP.

정글 몬스터 관련 주요 문제:  
**정글러 역할(laneRole='jungle')이 있음에도 AI 정글러의 행동이 단순 순회에 불과**하다. 정글 몬스터 처치 후 라인 지원(갱킹) 로직이 존재하지 않아, AI 정글러가 사실상 느린 미니언처럼 동작한다.

### 5-3. 에픽 드래곤 분석

- 등장: 5분마다 (5, 10, 15분)  
- 체력: 27,000 × scale, 공격력: 270 × scale  
- 처치 보상: 팀 전체 무작위 2가지 스탯 버프 (5분 지속), 400G

에픽 드래곤의 체력이 스케일(1→2→3)에 따라 27,000→54,000→81,000이 된다. 15분 에픽 드래곤(HP 81,000)은 일반적인 영웅 DPS로 처치 시간이 수분이 걸릴 수 있다. 이 보스에 집중하는 동안 팀의 라인 방치로 게임이 무너지는 딜레마가 발생한다.

**권장:** 15분 드래곤 처치 보상을 더 강하게 — 5분 버프 → 버프 유지 + 팀 전원 골드 1,000G 추가 보상

## 제6장. AI 인공지능 분석

### 6-1. AI 상태머신 구조

```
상태: RETREAT / ATTACK / TEAMFIGHT_JOIN / LANE
전환 조건:
- HP ≤ 35%: RETREAT (단, 극저체력 적 발견 시 ATTACK 유지)
- 근처에 전투 중인 아군 + 적 영웅: TEAMFIGHT_JOIN
- 적 존재: ATTACK
- 그 외: LANE
```

이 상태머신은 **반응형(reactive)**이지 **계획형(proactive)**이 아니다. MOBA AI에서 가장 중요한 요소인 **라인 우선순위 결정, 오브젝트 스쿼드 구성, 리콜-재진입 타이밍**이 전혀 없다.

### 6-2. AI 주요 문제점

**① 아이템 구매 무작위성 (가장 큰 문제)**

```javascript
aiShopAI(){
    let item=BASE_ITEMS[Math.floor(Math.random()*BASE_ITEMS.length)];
    if(this.gold >= item.cost) this.buyItem(item.id);
}
```

AI는 영웅 타입과 무관하게 **24개 아이템 중 무작위로 선택**한다. 근접 영웅인 크래그가 저격수 렌즈를 사고, 원거리 영웅 아리엘이 티아맷 도끼를 사는 상황이 발생한다. 하지만 `buyItem()` 내에서 타입 제한은 없으므로 구매는 가능하지만 스탯은 적용 안 된다 (applyStats에서 stat 조건 분기). 즉 **AI가 쓸모없는 아이템에 골드를 낭비한다.**

**수정 방법:**
```javascript
aiShopAI(){
    let heroType = HERO_TMPL[this.heroKey].type; // 'melee' or 'ranged'
    let eligible = BASE_ITEMS.filter(i => 
        i.heroType === heroType || i.heroType === 'magic' || i.heroType === 'common'
    );
    let item = eligible[Math.floor(Math.random() * eligible.length)];
    if(this.gold >= item.cost) this.buyItem(item.id);
}
```

**② AI 스킬 선택 완전 무작위**

```javascript
aiSelectSkill() {
    let available = PASSIVE_SKILLS.filter(s=>(this.passiveSkills[s.id]||0)<s.maxLv);
    let pick = available[Math.floor(Math.random()*available.length)];
    ...
}
```

AI는 영웅 타입 필터링도 없이 완전 무작위로 패시브를 선택한다. 원거리 AI 영웅이 근접 전용 패시브(fireRing, thornArmor)를 선택하는 사태가 발생한다.

**수정:**
```javascript
aiSelectSkill() {
    let pType = HERO_TMPL[this.heroKey].type;
    let available = PASSIVE_SKILLS.filter(s => {
        if ((this.passiveSkills[s.id]||0) >= s.maxLv) return false;
        if (pType === 'melee' && s.heroType === 'ranged') return false;
        if (pType === 'ranged' && s.heroType === 'melee') return false;
        return true;
    });
    ...
}
```
(플레이어의 showSkillSelection과 동일한 필터를 AI에도 적용)

**③ 라인 목적지 단순화**

```javascript
case 'LANE':
    if(this.laneRole === 'top')  { tx = 300;  ty = 300;  }
    if(this.laneRole === 'bot')  { tx = 2700; ty = 2700; }
    if(this.laneRole === 'mid')  { tx = 1500; ty = 1500; }
```

AI는 맵의 한 지점으로만 이동한다. 웨이포인트 기반 라인 패스 추종이 없어 AI가 지름길을 타거나 미니언 웨이브를 따라 이동하는 행동이 없다. 결과적으로 AI가 미니언보다 먼저 진입해 타워 어그로를 혼자 받는 상황이 잦다.

### 6-3. AI 타겟팅 평가

```javascript
score += (1.0 - (e.hp / e.maxHp)) * 400;  // 낮은 HP 우선
score += (1.0 - (d / 900)) * 300;          // 가까운 적 우선
if (e.type === 'hero') score += 250;        // 영웅 우선
if (tower && !미니언 탱킹) score -= 600;   // 타워 다이브 억제
```

타겟팅 로직 자체는 합리적이다. 낮은 체력 + 가까운 거리 + 영웅 우선이 올바른 방향. 타워 다이브 억제도 의도가 좋다. 다만 **미니언이 탱킹 중일 때 타워를 제외하는 로직**이 AI가 안전하게 타워를 쉽게 딜하도록 만들어, 타워의 전략적 가치를 떨어뜨린다.

**권장 변경:** 전술 회피 → `집탄 집중` (cd:22): 다음 기본 공격이 트루데미지 2배, 3초 후 자동 발동 (고위험 특화)

## 제8장. 조합 아이템(진화) 시스템 분석

### 8-1. 현황

14종의 진화 아이템 모두 `reqItem(아이템 7강) + reqPassive(패시브 스킬 Max)` 조건을 충족해야 한다.

### 8-2. 진단: 진화 조건의 루트 병목

전체 14종 중:
- 근접 아이템 조합 기반: 8종 (avalon_sword, reaper_blade, divine_shield, sunfire_cape, demonfire_blade, blood_thirster, oracle_glory, vampiric_cloak)
- 원거리 아이템 조합 기반: 4종 (frozen_heart, phantom_dagger, eye_of_storm, zeus_bracelet)
- 마법 기반: 2종 (archmage_staff, hourglass_fate)

원거리+마법 조합이 6종인데, 15명 영웅 중 원거리/마법 영웅이 8명(궁수, 조커, 암흑사제, 아리엘, 이스버그, 아칸, 실비아, 제피르)이다. 영웅 수 대비 진화 경로가 적다.

### 8-3. 진단: 오라클 영광 진화 경로 불일치

```javascript
{ id:'oracle_glory', reqItem:'steel_plate', reqPassive:'poisonCloud' }
```

steel_plate는 `heroType:'common'`(공용), poisonCloud는 `heroType:'melee'`(근접 전용). 원거리 영웅은 poisonCloud를 선택할 수 없으므로 **이 진화는 근접 영웅 전용**이지만 진화 아이템명과 설명이 이를 명시하지 않는다. 플레이어가 원거리로 steel_plate 7강을 달성한 후에야 진화 불가임을 알게 되는 실망스러운 경험이 발생한다.

## 마무리: 제작사에게 드리는 한 마디

운빨 아발론은 **"RNG 성장 + MOBA"라는 아직 아무도 제대로 만들지 않은 장르**를 브라우저에서 구현하고 있다. 컨셉 자체는 충분히 시장성이 있다.

지금 이 게임이 가진 가장 큰 적은 외부 경쟁이 아니라 **스스로 설정한 기술 부채**다. 9초 미니언, 흘러넘치는 골드, 사거리가 타워를 앞서는 원거리 영웅들이 만들어내는 "전략 없이도 되는 게임" 느낌이 반복 플레이 의욕을 꺾는다.

코드 품질은 높다. 구조는 잡혀 있다. 지금 필요한 건 기능 추가가 아니라 **기존 시스템의 수치와 로직 정비**다. 본 보고서의 Critical 항목 5개만 적용해도 게임은 한 단계 다른 느낌이 된다.
