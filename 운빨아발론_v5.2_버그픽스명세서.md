## 1. 화면 흔들림(Screen Shake) 완전 제거

### 전역 변수 삭제
```javascript
// 아래 줄 삭제
let screenShake = { x:0, y:0, intensity:0, duration:0 };
```

### shakeScreen 함수 삭제
```javascript
// 아래 함수 전체 삭제
function shakeScreen(intensity, duration) {
    screenShake.intensity = intensity;
    screenShake.duration = duration;
}
```

### gameLoop 내 screenShake 업데이트 블록 삭제
```javascript
// 아래 줄 삭제
if(screenShake.duration>0) { screenShake.duration-=dt; screenShake.x=rand(-screenShake.intensity,screenShake.intensity); screenShake.y=rand(-screenShake.intensity,screenShake.intensity); } else { screenShake.x=0; screenShake.y=0; }
```

### draw() 함수 내 screenShake 적용 부분 수정
```javascript
// 기존
ctx.translate(window.innerWidth/2 + screenShake.x * canvasDPR, window.innerHeight/2 + screenShake.y * canvasDPR);

// 교체
ctx.translate(window.innerWidth/2, window.innerHeight/2);
```

### useSkill() 내 모든 shakeScreen 호출 삭제
useSkill() 함수 내에 있는 아래 줄들을 **전부** 삭제:
```javascript
// 삭제 대상 (BERSERKER, VAMPIRE, THOR 등 여러 곳에 있는 shakeScreen 호출 전부)
shakeScreen(3, 0.15);
shakeScreen(5, 0.2);
shakeScreen(2, 0.15);
shakeScreen(4, 0.2);
shakeScreen(6, 0.25);
shakeScreen(7, 0.3);
```

### GS 전역 상태에서 hitStopTimer 관련 코드 정리
hitStopTimer는 screenShake와 함께 게임 느낌을 연출하는 요소인데, 화면 흔들림 제거와 함께 히트스톱도 제거 여부를 결정해야 함.

**히트스톱도 제거하려면** GS 선언에서 `hitStopTimer:0` 제거하고, 아래도 삭제:
```javascript
// applyRawDamage 내 삭제
if(isCrit || attacker === player) {
    GS.hitStopTimer = isCrit ? 0.04 : 0.02;
}

// gameLoop 내 삭제
if(!GS.paused && GS.hitStopTimer > 0) {
    GS.hitStopTimer -= dt;
    draw();
    requestAnimationFrame(gameLoop);
    return;
}
```

**히트스톱은 유지하려면** 위 두 블록을 그대로 두되, shakeScreen 호출만 지운다.

## 2. 모든 타격 기절 → 크리티컬 전용으로 제한

### 문제
`applyRawDamage()` 안에서 일반 공격에도 0.15초 기절을 적용. 공속 1.3짜리 영웅은 초당 1.3번 공격해서 상대를 사실상 계속 경직시킴. AI 전투가 슬로우모션처럼 보이는 원인.

### 수정
`applyRawDamage()` 내 기절/넉백 블록을 아래로 교체:

```javascript
// 기존 코드 (삭제)
if(!this.isBuilding && attacker && !attacker.isBuilding) {
    let angle = Math.atan2(this.y - attacker.y, this.x - attacker.x);
    let kbForce = isCrit ? 250 : 100;
    this.x += Math.cos(angle) * kbForce * 0.1;
    this.y += Math.sin(angle) * kbForce * 0.1;
    
    // 피격 경직 (0.15초~0.3초간 움직임 정지)
    this.stunTimer = Math.max(this.stunTimer || 0, isCrit ? 0.3 : 0.15);
}

// 교체 코드
if(!this.isBuilding && attacker && !attacker.isBuilding) {
    let angle = Math.atan2(this.y - attacker.y, this.x - attacker.x);
    // 넉백은 크리티컬에만 적용 (일반 공격은 넉백 없음)
    if(isCrit) {
        this.x += Math.cos(angle) * 20; // 크리티컬 넉백 약 20px
        this.y += Math.sin(angle) * 20;
        // 경직(stun)도 크리티컬 한정
        this.stunTimer = Math.max(this.stunTimer || 0, 0.2);
    }
}
```

## 3. 원거리 영웅 딜량(totalDmg) 투사체에서 누락

### 문제
`totalDmg`가 근접 공격 코드(`autoAttack()` 내 melee 브랜치)에서만 더해짐.
투사체(`Projectile.update()`)는 `attacker.totalDmg`를 건드리지 않아서,
궁수·네크로맨서·메카닉은 스코어보드에 딜량이 항상 0으로 표시됨.

### 수정

```javascript

```

## 4. 연속 레벨업 시 스킬 선택 화면 중복 출현

### 문제
`gainExp()` 내 `while` 루프에서 레벨업할 때마다 `setTimeout(() => this.showSkillSelection(), 500)`을 예약.
보스 처치 등으로 한 번에 2레벨 이상 오르면 스킬 선택 화면이 두 번 뜨거나 `GS.paused`가 꼬임.

### 수정 — 레벨업 큐 시스템 도입

**Hero 생성자에 추가:**
```javascript
// constructor 내에 추가
this.pendingSkillLevels = 0; // 대기 중인 스킬 선택 횟수
```

**`gainExp()` 함수 수정:**
```javascript
gainExp(amt){
    this.exp += amt;
    while(this.exp >= this.maxExp){
        this.exp -= this.maxExp;
        this.level++;
        this.maxExp = Math.floor(this.maxExp * 1.15);
        
        let stats = ['atk','hp','move','aspd'];
        let c = stats[Math.floor(Math.random()*stats.length)];
        let statMsg = '';
        if(c==='atk') { this.baseAtk+=6; statMsg='공격력 +6'; }
        if(c==='hp')  { this.baseMaxHp+=60; this.hp+=60; statMsg='체력 +60'; }
        if(c==='move'){ this.baseMoveSpd+=2.5; statMsg='이동속도 증가'; }
        if(c==='aspd'){ this.baseAspd+=0.12; statMsg='공격속도 증가'; }
        this.applyStats();
        
        if(this.isPlayer){
            addText(this.x, this.y-60, 'LEVEL UP!', '#fcd34d', 22);
            setTimeout(()=>addText(this.x, this.y-80, '운빨 스탯: '+statMsg+'!', '#a78bfa', 16), 300);
            playSFX('level_up');
            this.pendingSkillLevels++; // 큐에 추가 (바로 띄우지 않음)
        } else {
            this.aiSelectSkill();
        }
    }
    
    // 플레이어: 현재 스킬 선택 화면이 없을 때만 다음 큐 처리
    if(this.isPlayer && this.pendingSkillLevels > 0 && !GS.paused) {
        this.pendingSkillLevels--;
        setTimeout(() => this.showSkillSelection(), 400);
    }
}
```

## 5. 낙뢰 패시브 사운드 — 플레이어 한정으로 제한

### 문제
`spawnLightningEffect()` 안에 `playSFX('tower')` 호출이 있어서,
AI 영웅들이 낙뢰 패시브를 쓸 때도 계속 타워 공격음이 남.
현재 AI 영웅 9명이 낙뢰 패시브를 올릴 수 있어서 소리가 계속 튀어나옴.

### 수정
`triggerOnHitPassives()` 내 낙뢰 처리 블록에서 `spawnLightningEffect` 호출 시 attacker 정보를 전달:

```javascript
// spawnLightningEffect 함수 시그니처 변경
function spawnLightningEffect(x, y, isPlayerCaused = false) {
    for(let i=0;i<5;i++) particles.push({x:x+rand(-15,15), y:y-i*60, vx:rand(-20,20), vy:rand(-30,10), life:0.3, maxLife:0.3, color:'#fbbf24', size:rand(3,8), shape:'circle'});
    spawnParticles(x, y, '#fde68a', 12, 100, 0.3);
    if(isPlayerCaused) playSFX('tower'); // 플레이어 원인일 때만 소리
}
```

`triggerOnHitPassives()` 내 호출 위치 수정:
```javascript
// 기존
targets.forEach((t,idx)=>setTimeout(()=>{if(!t.isDead){t.applyRawDamage(this.atk*0.8,this);spawnLightningEffect(t.x,t.y);...}},idx*100));

targets.forEach((t,idx)=>setTimeout(()=>{if(!t.isDead){t.applyRawDamage(this.atk*0.8,this);spawnLightningEffect(t.x,t.y, this===player);...}},idx*100));
```

## 6. 좌향 공격 애니메이션 회전 반전 오류

### 문제
`drawBlockyHero()` 내에서 `dir < 0`일 때 `ctx.scale(-1, 1)` 좌우반전을 적용한 후,
`drawBody()` 안에서 `ctx.rotate(Math.PI/4)`를 실행하면 뒤집힌 좌표계에서 회전이 반대 방향으로 적용됨.
왼쪽을 보는 캐릭터가 공격할 때 몸이 앞(왼쪽)이 아닌 뒤(오른쪽)로 기울어짐.

### 수정
`drawBody()` 내 공격 회전 부분에서 dir을 고려해 회전 방향 보정:

```javascript
// drawBody 내 기존 코드 (삭제)
if(isAttacking && type !== 'vampire') {
    ctx.translate(x, y); ctx.rotate(Math.PI/4); ctx.translate(-x, -y);
}

// 교체: dir을 외부에서 받아서 방향 보정
// drawBody 함수 시그니처를 (skin, shirt, pants, dir) 로 변경하거나,
// dir을 drawBlockyHero 스코프에서 참조:

if(isAttacking && type !== 'vampire') {
    // dir < 0이면 반전된 좌표계에서의 회전이므로 부호 반전
    let rotDir = dir < 0 ? -1 : 1;
    ctx.translate(x, y);
    ctx.rotate(rotDir * Math.PI / 4); // 항상 앞쪽으로 기울어짐
    ctx.translate(-x, -y);
}
```

**단, drawBody는 drawBlockyHero 내부 함수이므로 dir 변수에 이미 접근 가능.** 클로저 스코프 활용해서 별도 파라미터 추가 없이 구현 가능.

각 영웅별 무기 `ctx.rotate` 포즈도 동일하게 적용:
- BERSERKER 무기: `ctx.rotate(Math.PI * 0.7 * rotDir)` 형태로
- THOR 망치: `ctx.rotate(Math.PI * 0.8 * rotDir)` 형태로
- NECROMANCER 지팡이: `ctx.rotate(Math.PI/2 * rotDir)` 형태로
- MECHANIC 총: `ctx.rotate(-Math.PI * 0.1 * rotDir)` 형태로

> `let rotDir = dir < 0 ? -1 : 1;` 를 `drawBlockyHero` 함수 상단에 한 번만 선언하고 모든 무기 rotate에 적용.

## 7. 미니언 타겟 우선순위 수정 (건물 > 미니언 > 영웅)

### 문제
`Minion.update()` 내 타겟팅이 150 거리 내 아무 적이나 공격.
미니언이 타워를 무시하고 영웅에게 달려드는 현상 발생.

### 수정
`Minion.update()` 내 타겟팅 로직 전면 교체:

```javascript
// 기존 타겟팅 코드 (삭제)
let target=null, minD=150;
entities.forEach(e=>{
    if(e.faction!==this.faction && !e.isDead){
        let d=dist(this,e);
        if(d<minD){ minD=d; target=e; }
    }
});

// 교체: 건물 > 미니언 > 영웅 우선순위
let closestBuilding = null, closestMinion = null, closestHero = null;
let dB = 120, dM = 100, dH = 120;
entities.forEach(e => {
    if(e.faction === this.faction || e.isDead) return;
    const d = dist(this, e);
    if((e.type==='tower' || e.type==='nexus_turret' || e.type==='nexus') && d < dB) {
        dB = d; closestBuilding = e;
    } else if(e.type === 'minion' && d < dM) {
        dM = d; closestMinion = e;
    } else if(e.type === 'hero' && d < dH) {
        dH = d; closestHero = e;
    }
});
let target = closestBuilding || closestMinion || closestHero;
```
